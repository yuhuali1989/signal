# SemDeDup：语义去重让 LLM 训练数据更高效

> **论文**: SemDeDup: Data-efficient learning at web-scale through semantic deduplication
> **作者**: Amro Abbas, Kushal Tirumala, Dániel Simig, Surya Ganguli, Ari S. Morcos (Meta FAIR)
> **发表**: ICLR 2023
> **arXiv**: 2303.09540

## 一句话总结

SemDeDup 利用预训练模型的语义嵌入对大规模数据集进行语义级去重，在 C4 和 LAION 数据集上证明：**去除 50% 语义重复数据后，模型性能不降反升**，训练效率提升 2x。

## 核心问题

传统去重方法（精确匹配、MinHash、n-gram）只能发现**表面重复**（文本/像素完全相同），但无法识别**语义重复**：
- 同一场景不同角度的照片
- 同一内容的不同表述
- 自动驾驶中同一路段连续帧的高度相似图像

这些语义重复数据占据大量存储和计算资源，却对模型学习贡献极低。

## 核心方法

### 1. 嵌入提取

```
原始数据 → 预训练模型 (CLIP / SigLIP) → 高维嵌入向量
```

- 图像数据：使用 CLIP ViT-L/14 提取 768 维嵌入
- 文本数据：使用 Sentence-BERT 或 CLIP 文本编码器

### 2. 聚类分组

```
嵌入向量 → K-Means 聚类 (K=50000) → 簇内排序
```

- 将数据按语义相似度分组
- 每个簇内的数据语义高度相关

### 3. 簇内去重

```
簇内数据 → 余弦相似度计算 → 阈值过滤 (ε=0.02~0.1)
```

- 对每个簇内的数据对计算余弦相似度
- 相似度 > (1 - ε) 的数据对，保留一个，删除另一个
- ε 越小越激进（去除更多数据）

### 4. 关键参数

| 参数 | 含义 | 推荐值 |
|------|------|--------|
| K (聚类数) | K-Means 簇数 | 50,000 |
| ε (阈值) | 去重激进程度 | 0.02~0.05 |
| 嵌入模型 | 语义表示质量 | CLIP ViT-L/14 |

## 核心实验结果

### LAION 图像数据集

| 去重比例 | 数据量 | ImageNet Acc | 训练时间 |
|---------|--------|-------------|---------|
| 0% (原始) | 233M | 63.2% | 1x |
| 25% | 175M | 63.5% (+0.3) | 0.75x |
| **50%** | **117M** | **63.1% (-0.1)** | **0.5x** |
| 75% | 58M | 61.8% (-1.4) | 0.25x |

### C4 文本数据集

| 去重比例 | 困惑度 (PPL) | 下游任务平均 |
|---------|-------------|------------|
| 0% (原始) | 18.2 | 72.1% |
| **50%** | **17.8 (-0.4)** | **72.5% (+0.4)** |

**关键发现**：去除 50% 语义重复数据后，模型性能基本不变甚至略有提升，但训练成本减半。

## 与传统去重方法对比

| 方法 | 去重粒度 | 可去除比例 | 性能影响 | 计算成本 |
|------|---------|-----------|---------|---------|
| 精确匹配 | 完全相同 | ~5% | 无 | 极低 |
| MinHash (n-gram) | 近似文本 | ~15% | 无 | 低 |
| URL 去重 | 相同来源 | ~10% | 无 | 极低 |
| **SemDeDup** | **语义相似** | **~50%** | **无/正向** | **中** |

## 在自动驾驶数据闭环中的应用

SemDeDup 对自动驾驶数据特别有价值：

1. **连续帧去重**：同一路段连续采集的帧高度相似，语义去重可去除 60-80% 冗余帧
2. **多车重复路段**：不同车辆经过同一路段的数据语义重复
3. **静态场景过滤**：停车场/等红灯等低信息量场景
4. **跨数据集去重**：nuScenes + OpenDV-2K 之间的重叠场景

### 自动驾驶去重 Pipeline

```
6路相机帧 → CLIP 编码 → Milvus 向量检索 → 语义相似度 > 0.95 → 标记为重复
                                           → 0.85~0.95 → 保留但降低采样权重
                                           → < 0.85 → 保留（新场景）
```

## 局限性

1. **嵌入质量依赖**：去重效果取决于预训练模型的语义表示质量
2. **计算开销**：大规模数据集的嵌入提取和聚类需要 GPU 资源
3. **阈值敏感**：ε 参数需要针对不同数据集调优
4. **长尾场景风险**：过度去重可能误删稀有但重要的长尾场景

## 影响力

SemDeDup 开创了**语义级数据去重**范式，直接影响了：
- Meta 的 Llama 3 预训练数据处理 pipeline
- Hugging Face FineWeb 的去重策略
- 自动驾驶数据闭环中的数据质量管理
