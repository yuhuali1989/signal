# 第 1 章：Transformer 架构深度剖析

> 选自《大语言模型从入门到前沿》

## 1.1 从 RNN 到 Transformer：一场必然的革命

在 Transformer 出现之前，序列建模主要依赖 RNN 及其变体（LSTM、GRU）。这些模型的核心问题是**顺序计算**——每个时间步必须等待前一步完成，无法并行化。

| 模型 | 并行性 | 长距离依赖 | 训练速度 | 代表工作 |
|------|--------|-----------|---------|---------|
| RNN | ❌ | 差（梯度消失） | 慢 | Elman (1990) |
| LSTM | ❌ | 中等 | 慢 | Hochreiter (1997) |
| GRU | ❌ | 中等 | 较慢 | Cho (2014) |
| Transformer | ✅ | 优秀 | 快 | Vaswani (2017) |

2017 年 Google 发表 "Attention Is All You Need"，Transformer 架构彻底改变了 NLP，后来更扩展到视觉、语音、多模态等领域。

## 1.2 Self-Attention：Transformer 的灵魂

Self-Attention 的核心思想是让序列中的每个位置都能直接关注其他所有位置，无需通过中间状态传递信息。

### 数学定义

给定输入序列 $X \in \mathbb{R}^{n \times d}$，通过三个线性变换得到 Query、Key、Value：

$$Q = XW_Q, \quad K = XW_K, \quad V = XW_V$$

其中 $W_Q, W_K \in \mathbb{R}^{d \times d_k}$，$W_V \in \mathbb{R}^{d \times d_v}$。

注意力计算公式：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

$\sqrt{d_k}$ 是缩放因子，防止内积值过大导致 softmax 梯度消失。

### PyTorch 实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class ScaledDotProductAttention(nn.Module):
    def __init__(self, d_k):
        super().__init__()
        self.scale = d_k ** 0.5

    def forward(self, Q, K, V, mask=None):
        # Q, K, V: (batch, seq_len, d_k)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / self.scale
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        attn_weights = F.softmax(scores, dim=-1)
        output = torch.matmul(attn_weights, V)
        return output, attn_weights
```

### 复杂度分析

Self-Attention 的时间和空间复杂度均为 $O(n^2 d)$，其中 $n$ 是序列长度。这是 Transformer 处理长序列时的主要瓶颈，催生了后来的 FlashAttention、稀疏注意力等优化。

## 1.3 Multi-Head Attention：多角度观察

单个注意力头只能学到一种注意力模式。Multi-Head Attention 通过 $h$ 个并行的注意力头，让模型同时关注不同子空间的信息：

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \dots, \text{head}_h)W_O$$

其中每个头独立计算：

$$\text{head}_i = \text{Attention}(QW_Q^i, KW_K^i, VW_V^i)$$

```python
class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        assert d_model % n_heads == 0
        self.d_k = d_model // n_heads
        self.n_heads = n_heads

        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, Q, K, V, mask=None):
        batch_size = Q.size(0)

        # 线性变换并分头: (batch, seq, d_model) -> (batch, n_heads, seq, d_k)
        Q = self.W_q(Q).view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)
        K = self.W_k(K).view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)
        V = self.W_v(V).view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)

        # 缩放点积注意力
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        attn = F.softmax(scores, dim=-1)
        output = torch.matmul(attn, V)

        # 合并多头: (batch, n_heads, seq, d_k) -> (batch, seq, d_model)
        output = output.transpose(1, 2).contiguous().view(batch_size, -1, self.n_heads * self.d_k)
        return self.W_o(output)
```

**GPT-3 的参数**：$d_{model} = 12288$，$h = 96$，$d_k = 128$。

## 1.4 FlashAttention：让长序列成为可能

标准 Attention 需要 $O(n^2)$ 的 HBM（显存）存储完整的 attention 矩阵。FlashAttention 的核心洞察是：**通过分块计算（tiling）避免物化完整的 $n \times n$ 矩阵**。

### GPU 存储层次

```
┌─────────────────────────────────────┐
│         HBM (显存) ~80GB            │  ← 带宽 ~3TB/s
│     存放 Q, K, V, Output 矩阵      │
├─────────────────────────────────────┤
│         SRAM (片上) ~20MB           │  ← 带宽 ~19TB/s
│     分块计算 attention              │
└─────────────────────────────────────┘
```

FlashAttention 的关键技术：

1. **分块加载**：将 Q, K, V 切成小块（block），每次只在 SRAM 中计算一个 block 的 attention
2. **在线 softmax**：使用 Milakov & Gimelshein (2018) 的在线算法，无需完整存储 $e^{x_i}$
3. **重计算代替存储**：反向传播时重算 attention matrix，用计算换显存

$$m_j = \max(m_{j-1}, \max(S_{:,j})), \quad \ell_j = e^{m_{j-1} - m_j} \ell_{j-1} + \text{rowsum}(e^{S_{:,j} - m_j})$$

| 方法 | 显存 | IO 复杂度 | 速度提升 |
|------|------|----------|---------|
| Standard Attention | $O(n^2)$ | $O(n^2 d + n^2)$ | 1× |
| FlashAttention | $O(n)$ | $O(n^2 d^2 / M)$ | 2-4× |
| FlashAttention-2 | $O(n)$ | 优化并行 | 3-7× |
| FlashAttention-3 | $O(n)$ | Hopper 优化 | 1.5-2× (vs FA2) |

## 1.5 Feed-Forward Network (SwiGLU)

Transformer 中每个注意力层后接一个前馈网络（FFN）。原始 Transformer 用 ReLU：

$$\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2$$

现代 LLM 几乎都使用 SwiGLU 激活（Shazeer 2020）：

$$\text{SwiGLU}(x) = (\text{Swish}(xW_1) \odot xW_3) W_2$$

其中 Swish 也叫 SiLU：$\text{Swish}(x) = x \cdot \sigma(x)$。

```python
class SwiGLU(nn.Module):
    def __init__(self, d_model, d_ff):
        super().__init__()
        self.W1 = nn.Linear(d_model, d_ff, bias=False)
        self.W3 = nn.Linear(d_model, d_ff, bias=False)  # Gate
        self.W2 = nn.Linear(d_ff, d_model, bias=False)

    def forward(self, x):
        return self.W2(F.silu(self.W1(x)) * self.W3(x))
```

SwiGLU 相比 ReLU 的优势：在相同计算量下，模型质量提升约 1-2%（Llama 论文验证）。

## 1.6 RMSNorm：更快的归一化

LayerNorm 需要计算均值和方差，而 RMSNorm（Zhang & Sennrich 2019）只计算均方根，速度更快 10-15%：

$$\text{RMSNorm}(x) = \frac{x}{\sqrt{\frac{1}{d}\sum_{i=1}^{d}x_i^2 + \epsilon}} \cdot \gamma$$

```python
class RMSNorm(nn.Module):
    def __init__(self, dim, eps=1e-6):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(dim))

    def forward(self, x):
        rms = torch.sqrt(torch.mean(x ** 2, dim=-1, keepdim=True) + self.eps)
        return x / rms * self.weight
```

## 1.7 位置编码：RoPE

Transformer 本身没有位置感知，需要额外注入位置信息。

**绝对位置编码**（原始 Transformer）：

$$PE_{(pos, 2i)} = \sin(pos / 10000^{2i/d}), \quad PE_{(pos, 2i+1)} = \cos(pos / 10000^{2i/d})$$

**旋转位置编码 RoPE**（Su et al. 2021）：

RoPE 将位置信息编码为旋转矩阵，直接作用于 Q 和 K 向量：

$$f(x_m, m) = R_{\Theta,m} x_m$$

其中旋转矩阵 $R_{\Theta,m}$ 将向量按维度对两两分组，每组施加角度为 $m\theta_i$ 的旋转。

RoPE 的核心优势是**相对位置信息自然嵌入内积**：

$$\langle f(q, m), f(k, n) \rangle = g(q, k, m - n)$$

```python
def precompute_freqs_cis(dim, end, theta=10000.0):
    """预计算 RoPE 频率"""
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2).float() / dim))
    t = torch.arange(end)
    freqs = torch.outer(t, freqs)
    return torch.polar(torch.ones_like(freqs), freqs)  # e^{i*theta}

def apply_rotary_emb(xq, xk, freqs_cis):
    """应用 RoPE 到 Q 和 K"""
    xq_ = torch.view_as_complex(xq.float().reshape(*xq.shape[:-1], -1, 2))
    xk_ = torch.view_as_complex(xk.float().reshape(*xk.shape[:-1], -1, 2))
    xq_out = torch.view_as_real(xq_ * freqs_cis).flatten(-2)
    xk_out = torch.view_as_real(xk_ * freqs_cis).flatten(-2)
    return xq_out.type_as(xq), xk_out.type_as(xk)
```

## 1.8 完整 Transformer Decoder Block

将以上组件组装为一个完整的 Decoder Block：

```python
class TransformerBlock(nn.Module):
    def __init__(self, d_model, n_heads, d_ff, norm_eps=1e-6):
        super().__init__()
        self.attention = MultiHeadAttention(d_model, n_heads)
        self.feed_forward = SwiGLU(d_model, d_ff)
        self.attention_norm = RMSNorm(d_model, norm_eps)
        self.ffn_norm = RMSNorm(d_model, norm_eps)

    def forward(self, x, mask=None):
        # Pre-Norm: norm → attention → residual
        h = x + self.attention(self.attention_norm(x), self.attention_norm(x),
                               self.attention_norm(x), mask)
        # Pre-Norm: norm → FFN → residual
        out = h + self.feed_forward(self.ffn_norm(h))
        return out
```

注意现代 LLM 都使用 **Pre-Norm**（先归一化再计算），而非原始 Transformer 的 Post-Norm。Pre-Norm 训练更稳定，尤其在深层网络中。

## 1.10 从 MHA 到 MLA：注意力的四代演进

**MHA（Multi-Head Attention）**：每个头独立维护 K、V cache，显存 $O(n \cdot h \cdot d_k)$

**MQA（Multi-Query Attention）**：所有头共享一份 K、V，显存减少为 $O(n \cdot d_k)$，但质量下降

**GQA（Grouped-Query Attention）**：折中方案，每组几个头共享 K、V：

$$\text{KV groups} = h / g, \quad g \in \{1, h\}$$

Llama 2 70B 用 8 组 GQA，在质量和效率间取得平衡。

**MLA（Multi-head Latent Attention）**：DeepSeek-V2/V3 的创新，将 KV 压缩到低维潜在空间：

$$c_t^{KV} = W^{DKV} h_t \in \mathbb{R}^{d_c}, \quad d_c \ll d_h \cdot n_h$$

KV Cache 从 $n_h \cdot d_h = 128 \times 128 = 16384$ 维压缩到 $d_c = 512$ 维，**压缩比 32×**。
