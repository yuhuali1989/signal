# 第 6 章：生成模型——从 AE 到扩散模型

> 选自《大语言模型从入门到前沿》

生成模型（Generative Model）的目标是**学习数据的分布 $p(x)$，并从中采样生成新样本**。从最早的自编码器到今天的扩散模型，这条演进路线是理解 FLUX.1、Sora、Kling 等现代生成系统的基础。

```
技术演进路线：
AE（2006）→ VAE（2013）→ GAN（2014）→ DDPM（2020）
→ LDM/Stable Diffusion（2022）→ Flow Matching（2022）
→ DiT（2023）→ FLUX.1/SD3（2024）→ Sora/Kling/Veo（2024-2025）
```

---

## 6.1 自编码器（AE）：压缩与重建

### 6.1.1 核心思想

自编码器（Autoencoder，AE）是生成模型的起点。它的目标很简单：**用一个低维向量（潜在表示）来重建原始输入**。

```
输入 x → [编码器 Encoder] → 潜在向量 z → [解码器 Decoder] → 重建 x̂
```

训练目标是最小化重建误差：

$$\mathcal{L}_{AE} = \| x - \hat{x} \|^2 = \| x - D(E(x)) \|^2$$

```python
import torch
import torch.nn as nn

class Autoencoder(nn.Module):
    """最简单的全连接自编码器"""
    def __init__(self, input_dim=784, latent_dim=32):
        super().__init__()
        # 编码器：784 → 256 → 32
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 256),
            nn.ReLU(),
            nn.Linear(256, latent_dim)
        )
        # 解码器：32 → 256 → 784
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 256),
            nn.ReLU(),
            nn.Linear(256, input_dim),
            nn.Sigmoid()  # 输出归一化到 [0,1]
        )
    
    def forward(self, x):
        z = self.encoder(x)      # 压缩
        x_hat = self.decoder(z)  # 重建
        return x_hat, z

# 训练
model = Autoencoder()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

for x_batch in dataloader:
    x_hat, z = model(x_batch)
    loss = nn.MSELoss()(x_hat, x_batch)  # 重建损失
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

### 6.1.2 AE 的问题：潜空间不连续

AE 能压缩和重建，但**不能生成新样本**。原因是潜空间 $z$ 没有任何约束——不同样本的潜在向量可能分布在空间中的孤立点，两点之间的插值解码出来是无意义的噪声。

```
AE 潜空间（不连续）：
  ●  ●        ●
      ↑
   这里是空洞，解码出垃圾
  ●       ●  ●
```

**核心问题**：AE 的潜空间没有概率意义，无法从中随机采样生成有意义的新样本。

## 6.2 变分自编码器（VAE）：让潜空间有概率意义

### 6.2.1 核心思想

VAE（Variational Autoencoder，Kingma & Welling，2013）的关键创新：**不再把输入编码为一个固定向量，而是编码为一个概率分布**。

$$q_\phi(z|x) = \mathcal{N}(\mu_\phi(x),\ \sigma^2_\phi(x) \cdot I)$$

编码器输出均值 $\mu$ 和方差 $\sigma^2$，然后从这个分布中**采样** $z$，再解码重建。

```
输入 x → [编码器] → (μ, σ²) → 采样 z ~ N(μ, σ²I) → [解码器] → x̂
                                    ↑
                              重参数化技巧（可反向传播）
```

### 6.2.2 ELBO：VAE 的训练目标

VAE 的目标是最大化数据的对数似然 $\log p(x)$。通过变分推断，推导出**证据下界（ELBO，Evidence Lower BOund）**：

$$\log p(x) \geq \underbrace{\mathbb{E}_{q_\phi(z|x)}[\log p_\theta(x|z)]}_{\text{重建项}} - \underbrace{D_{KL}(q_\phi(z|x) \| p(z))}_{\text{KL 正则项}}$$

**两项的直觉理解**：

| 项 | 数学含义 | 直觉 |
|---|---------|------|
| **重建项** | $\mathbb{E}[\log p_\theta(x\|z)]$ | 解码器能从 $z$ 重建出 $x$（重建质量） |
| **KL 正则项** | $D_{KL}(q_\phi(z\|x) \| p(z))$ | 编码器输出的分布要接近标准正态 $\mathcal{N}(0,I)$（潜空间规整） |

KL 正则项强制潜空间变得**连续且规整**——不同样本的潜在分布相互重叠，插值有意义，随机采样也有意义。

**高斯分布的 KL 散度闭合解**：

$$D_{KL}(\mathcal{N}(\mu, \sigma^2) \| \mathcal{N}(0, 1)) = \frac{1}{2}\sum_{j=1}^{d}\left(\mu_j^2 + \sigma_j^2 - \log\sigma_j^2 - 1\right)$$

### 6.2.3 重参数化技巧（Reparameterization Trick）

**问题**：采样操作 $z \sim \mathcal{N}(\mu, \sigma^2)$ 是随机的，梯度无法通过它反向传播。

**解决方案**：把随机性"外移"——

$$z = \mu + \sigma \odot \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)$$

现在 $\epsilon$ 是外部随机噪声，$z$ 关于 $\mu$ 和 $\sigma$ 是**可微的**，梯度可以正常反向传播。

```python
class VAE(nn.Module):
    """变分自编码器完整实现"""
    def __init__(self, input_dim=784, latent_dim=32):
        super().__init__()
        # 编码器：输出 μ 和 log σ²（用 log 保证数值稳定）
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 256), nn.ReLU()
        )
        self.fc_mu = nn.Linear(256, latent_dim)       # 均值头
        self.fc_logvar = nn.Linear(256, latent_dim)   # log方差头
        
        # 解码器
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 256), nn.ReLU(),
            nn.Linear(256, input_dim), nn.Sigmoid()
        )
    
    def encode(self, x):
        h = self.encoder(x)
        mu = self.fc_mu(h)
        logvar = self.fc_logvar(h)
        return mu, logvar
    
    def reparameterize(self, mu, logvar):
        """重参数化技巧：z = μ + σ * ε，ε ~ N(0,I)"""
        if self.training:
            std = torch.exp(0.5 * logvar)   # σ = exp(log σ² / 2)
            eps = torch.randn_like(std)      # ε ~ N(0, I)
            return mu + std * eps            # 可微的采样
        else:
            return mu  # 推理时直接用均值
    
    def decode(self, z):
        return self.decoder(z)
    
    def forward(self, x):
        mu, logvar = self.encode(x)
        z = self.reparameterize(mu, logvar)
        x_hat = self.decode(z)
        return x_hat, mu, logvar
    
    def loss(self, x, x_hat, mu, logvar, beta=1.0):
        """ELBO 损失 = 重建损失 + β * KL 散度"""
        # 重建损失（BCE 对应伯努利分布假设）
        recon_loss = nn.BCELoss(reduction='sum')(x_hat, x)
        
        # KL 散度（闭合解）
        kl_loss = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
        
        return recon_loss + beta * kl_loss

# 生成新样本：直接从标准正态采样
def generate(vae, n_samples=16):
    z = torch.randn(n_samples, 32)  # 从 N(0,I) 采样
    with torch.no_grad():
        samples = vae.decode(z)
    return samples
```

### 6.2.4 β-VAE：控制潜空间解耦程度

β-VAE（Higgins et al., 2017）在 KL 项前加权重 $\beta > 1$，强制更强的正则化，让潜空间的每个维度对应一个独立的语义因子（解耦表示）：

$$\mathcal{L}_{\beta\text{-VAE}} = \mathbb{E}[\log p_\theta(x|z)] - \beta \cdot D_{KL}(q_\phi(z|x) \| p(z))$$

- $\beta = 1$：标准 VAE
- $\beta > 1$：更强解耦，但重建质量下降
- $\beta < 1$：更好重建，但潜空间不够规整

### 6.2.5 VAE 的局限：生成质量模糊

VAE 的重建损失（MSE/BCE）倾向于生成**模糊的平均图像**，因为它对所有像素误差一视同仁，而不关注感知质量。这是 VAE 在图像生成质量上不如 GAN 和扩散模型的根本原因。

## 6.3 GAN：对抗训练与模式崩溃

### 6.3.1 核心思想

GAN（Generative Adversarial Network，Goodfellow et al., 2014）用**博弈论**思路解决生成问题：

```
生成器 G：噪声 z → 假图像 G(z)（想骗过判别器）
判别器 D：图像 x → 真/假概率 D(x)（想识别真假）

两者对抗训练，最终 G 生成的图像让 D 无法区分真假
```

**训练目标**（minimax 博弈）：

$$\min_G \max_D \mathbb{E}_{x \sim p_{data}}[\log D(x)] + \mathbb{E}_{z \sim p_z}[\log(1 - D(G(z)))]$$

### 6.3.2 GAN 的核心问题

| 问题 | 描述 | 影响 |
|------|------|------|
| **模式崩溃** | G 只学会生成少数几种样本 | 多样性差 |
| **训练不稳定** | G 和 D 的训练需要精心平衡 | 难以收敛 |
| **评估困难** | 没有明确的损失函数衡量生成质量 | 调参困难 |
| **无法推断** | 给定图像无法得到对应的 z | 不支持编辑 |

正是这些问题，推动了扩散模型的兴起。

## 6.4 DDPM：去噪扩散概率模型

### 6.4.1 核心思想

DDPM（Denoising Diffusion Probabilistic Models，Ho et al., 2020）的核心思想：**学习逆转一个逐步加噪的过程**。

```
前向过程（加噪，固定，无需学习）：
x₀ → x₁ → x₂ → ... → xT ≈ N(0, I)
（原始图像）                （纯噪声）

反向过程（去噪，模型学习）：
xT → xT-1 → ... → x₁ → x₀
（纯噪声）              （生成图像）
```

### 6.4.2 前向过程（加噪）

每一步向图像中加入少量高斯噪声：

$$q(x_t | x_{t-1}) = \mathcal{N}(x_t;\ \sqrt{1-\beta_t}\, x_{t-1},\ \beta_t I)$$

其中 $\beta_t$ 是噪声调度（noise schedule），从小到大（如 $\beta_1 = 10^{-4}$ 到 $\beta_T = 0.02$）。

**关键性质**：可以直接从 $x_0$ 一步跳到任意时刻 $x_t$（无需逐步计算）：

$$q(x_t | x_0) = \mathcal{N}(x_t;\ \sqrt{\bar{\alpha}_t}\, x_0,\ (1-\bar{\alpha}_t) I)$$

即：

$$x_t = \sqrt{\bar{\alpha}_t}\, x_0 + \sqrt{1-\bar{\alpha}_t}\, \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)$$

其中 $\bar{\alpha}_t = \prod_{s=1}^{t}(1-\beta_s)$，随 $t$ 增大趋向 0（图像信号消失，变为纯噪声）。

```python
def q_sample(x0, t, noise_schedule):
    """前向加噪：从 x0 直接采样 xt"""
    alpha_bar = noise_schedule.alpha_bar[t]  # ᾱt
    noise = torch.randn_like(x0)
    
    # xt = √ᾱt * x0 + √(1-ᾱt) * ε
    xt = alpha_bar.sqrt() * x0 + (1 - alpha_bar).sqrt() * noise
    return xt, noise  # 返回 xt 和真实噪声（训练时用）
```

### 6.4.3 反向过程（去噪）

反向过程是一个马尔可夫链，每步从 $x_t$ 预测 $x_{t-1}$：

$$p_\theta(x_{t-1} | x_t) = \mathcal{N}(x_{t-1};\ \mu_\theta(x_t, t),\ \Sigma_\theta(x_t, t))$$

**关键推导**：DDPM 不直接预测 $\mu_\theta$，而是训练模型 $\epsilon_\theta$ 预测加入的噪声 $\epsilon$，然后用噪声推导出均值：

$$\mu_\theta(x_t, t) = \frac{1}{\sqrt{\alpha_t}}\left(x_t - \frac{\beta_t}{\sqrt{1-\bar{\alpha}_t}} \epsilon_\theta(x_t, t)\right)$$

### 6.4.4 训练目标

DDPM 的训练目标极其简洁——**预测噪声**：

$$\mathcal{L}_{simple} = \mathbb{E}_{x_0, \epsilon, t}\left[\| \epsilon - \epsilon_\theta(x_t, t) \|^2\right]$$

```python
def ddpm_train_step(model, x0, noise_schedule):
    """DDPM 单步训练"""
    batch_size = x0.shape[0]
    
    # 1. 随机采样时间步 t
    t = torch.randint(0, noise_schedule.T, (batch_size,))
    
    # 2. 采样噪声
    epsilon = torch.randn_like(x0)
    
    # 3. 前向加噪：得到 xt
    xt, _ = q_sample(x0, t, noise_schedule)
    
    # 4. 模型预测噪声
    epsilon_pred = model(xt, t)
    
    # 5. 损失：预测噪声 vs 真实噪声
    loss = F.mse_loss(epsilon_pred, epsilon)
    return loss

def ddpm_sample(model, noise_schedule, shape, steps=1000):
    """DDPM 采样（反向去噪）"""
    x = torch.randn(shape)  # 从纯噪声开始
    
    for t in reversed(range(steps)):
        t_batch = torch.full((shape[0],), t)
        
        # 预测噪声
        epsilon_pred = model(x, t_batch)
        
        # 计算均值
        alpha = noise_schedule.alpha[t]
        alpha_bar = noise_schedule.alpha_bar[t]
        beta = noise_schedule.beta[t]
        
        mu = (x - beta / (1 - alpha_bar).sqrt() * epsilon_pred) / alpha.sqrt()
        
        # 加入随机噪声（t > 0 时）
        if t > 0:
            sigma = beta.sqrt()
            x = mu + sigma * torch.randn_like(x)
        else:
            x = mu
    
    return x
```

### 6.4.5 DDPM 的问题：推理慢

DDPM 需要 **T=1000 步**反向去噪，每步都要跑一次神经网络，推理极慢（在 GPU 上生成一张图需要数十秒）。

**DDIM（Denoising Diffusion Implicit Models，Song et al., 2020）** 解决了这个问题：通过非马尔可夫采样，可以用 **50 步甚至 10 步**完成采样，质量几乎不变。

## 6.5 潜空间扩散模型（LDM）：在压缩空间中扩散

### 6.5.1 核心思想

直接在像素空间做扩散（如 DDPM）计算量巨大——一张 512×512 的图像有 786,432 个像素，每步都要处理这么大的张量。

**LDM（Latent Diffusion Model，Rombach et al., 2022）** 的核心创新：**先用 VAE 把图像压缩到低维潜空间，再在潜空间做扩散**。

```
训练阶段：
图像 x [512×512×3]
    ↓ VAE Encoder（压缩 8×）
潜变量 z [64×64×4]   ← 在这里做扩散！
    ↓ 扩散模型训练
    ↓ VAE Decoder（解压）
重建图像 x̂

推理阶段：
纯噪声 z_T [64×64×4]
    ↓ 扩散模型去噪（T步）
潜变量 z_0 [64×64×4]
    ↓ VAE Decoder
生成图像 [512×512×3]
```

### 6.5.2 为什么有效？

VAE 的编码器把图像压缩到潜空间，**去除了感知上不重要的高频细节**，保留了语义信息。扩散模型在这个低维空间工作：

- 计算量降低 **64 倍**（8×8 空间压缩）
- 训练速度提升 **4-8 倍**
- 生成质量不降反升（潜空间更平滑，扩散更容易学习）

```python
class LatentDiffusionModel(nn.Module):
    """潜空间扩散模型（Stable Diffusion 风格）"""
    def __init__(self):
        # 预训练 VAE（冻结权重）
        self.vae = AutoencoderKL.from_pretrained("stabilityai/sd-vae-ft-mse")
        self.vae.requires_grad_(False)
        
        # 扩散模型（在潜空间工作）
        self.unet = UNet2DConditionModel(
            in_channels=4,    # VAE 潜空间通道数
            out_channels=4,
            cross_attention_dim=768  # 文本 embedding 维度
        )
        
        # 文本编码器
        self.text_encoder = CLIPTextModel.from_pretrained("openai/clip-vit-large-patch14")
    
    def encode_image(self, x):
        """图像 → 潜变量（训练时用）"""
        with torch.no_grad():
            # VAE 编码，乘以缩放因子（标准化潜空间）
            z = self.vae.encode(x).latent_dist.sample() * 0.18215
        return z
    
    def decode_latent(self, z):
        """潜变量 → 图像（推理时用）"""
        with torch.no_grad():
            x = self.vae.decode(z / 0.18215).sample
        return x
    
    def train_step(self, images, text_prompts):
        # 1. 编码图像到潜空间
        z0 = self.encode_image(images)
        
        # 2. 编码文本条件
        text_emb = self.text_encoder(text_prompts).last_hidden_state
        
        # 3. 随机时间步 + 加噪
        t = torch.randint(0, 1000, (z0.shape[0],))
        noise = torch.randn_like(z0)
        zt = self.noise_scheduler.add_noise(z0, noise, t)
        
        # 4. UNet 预测噪声（以文本为条件）
        noise_pred = self.unet(zt, t, encoder_hidden_states=text_emb).sample
        
        # 5. 损失
        return F.mse_loss(noise_pred, noise)
```

### 6.5.3 VAE 在 LDM 中的角色

LDM 中的 VAE 与生成式 VAE 有所不同——它更像一个**高质量的有损压缩器**：

| 角色 | 生成式 VAE | LDM 中的 VAE |
|------|-----------|-------------|
| 目标 | 生成新样本 | 压缩/解压图像 |
| 潜空间 | 随机采样 | 确定性编码（推理时） |
| 训练 | ELBO（重建+KL） | 重建+感知损失+对抗损失 |
| 通道数 | 任意 | 通常 4 或 16 |
| 压缩比 | 任意 | 通常 8× 空间压缩 |

## 6.6 Classifier-Free Guidance（CFG）：条件生成的核心技术

### 6.6.1 背景：条件生成的挑战

文生图的核心是**条件生成**：给定文本描述 $c$，生成符合描述的图像。

早期方案是**Classifier Guidance**（Dhariwal & Nichol, 2021）：训练一个额外的分类器 $p_\phi(c|x_t)$，用其梯度引导扩散过程。但这需要额外训练分类器，且分类器要在噪声图像上工作，工程复杂。

### 6.6.2 CFG 的核心思想

**Classifier-Free Guidance（Ho & Salimans, 2022）** 的优雅解决方案：**不需要额外分类器，只需训练一个同时支持有条件和无条件生成的模型**。

训练时，随机以概率 $p_{uncond}$（通常 10-20%）将条件 $c$ 替换为空条件 $\emptyset$（空字符串或零向量）：

```python
def train_step_cfg(model, x0, condition, p_uncond=0.1):
    """CFG 训练：随机丢弃条件"""
    # 随机将部分样本的条件替换为空条件
    mask = torch.rand(x0.shape[0]) < p_uncond
    condition_input = condition.clone()
    condition_input[mask] = EMPTY_CONDITION  # 空条件（空字符串 embedding）
    
    # 正常训练
    t = torch.randint(0, T, (x0.shape[0],))
    noise = torch.randn_like(x0)
    xt = add_noise(x0, noise, t)
    
    noise_pred = model(xt, t, condition_input)
    return F.mse_loss(noise_pred, noise)
```

### 6.6.3 CFG 推理：放大条件信号

推理时，同时运行**有条件**和**无条件**两次前向传播，然后用 guidance scale $w$ 放大条件信号：

$$\tilde{\epsilon}_\theta(x_t, t, c) = \epsilon_\theta(x_t, t, \emptyset) + w \cdot \underbrace{(\epsilon_\theta(x_t, t, c) - \epsilon_\theta(x_t, t, \emptyset))}_{\text{条件方向}}$$

**直觉理解**：

```
无条件预测：模型"自由发挥"的方向
有条件预测：模型"按文本描述"的方向
差值：文本条件对生成方向的影响

CFG = 无条件方向 + w × 条件影响
     ↑ 基础方向    ↑ 放大条件信号
```

```python
def cfg_sample(model, xt, t, condition, guidance_scale=7.5):
    """CFG 采样：有条件 + 无条件双路推理"""
    # 有条件预测
    noise_cond = model(xt, t, condition)
    
    # 无条件预测（空条件）
    noise_uncond = model(xt, t, EMPTY_CONDITION)
    
    # CFG 公式：放大条件信号
    noise_guided = noise_uncond + guidance_scale * (noise_cond - noise_uncond)
    
    return noise_guided
```

### 6.6.4 Guidance Scale 的影响

| guidance_scale (w) | 效果 | 典型场景 |
|:-----------------:|------|---------|
| 1.0 | 等同于无条件生成，多样性最高 | 探索性生成 |
| 3-5 | 轻度引导，多样性和质量平衡 | 艺术创作 |
| **7-8** | **标准设置，质量和多样性最佳平衡** | **日常使用** |
| 10-15 | 强引导，严格遵循文本，但可能过饱和 | 精确控制 |
| >20 | 过度引导，图像失真、颜色过饱和 | 不推荐 |

**CFG 的代价**：每步推理需要运行两次模型（有条件 + 无条件），推理速度减半。这是 FLUX.1-schnell 等模型优化的重点之一（通过 Guidance Distillation 将双路合并为单路）。

### 6.6.5 为什么 CFG 有效？贝叶斯视角

从贝叶斯角度，CFG 等价于在对数概率空间中放大条件信号：

$$\log \tilde{p}(x_t | c) \propto \log p(x_t) + (w+1) \cdot \log p(c | x_t)$$

即：CFG 等价于用一个**隐式分类器** $p(c|x_t)^{w+1}$ 引导生成，guidance scale 越大，对条件的"坚持"越强。

## 6.7 Flow Matching：更直接的训练目标

### 6.7.1 扩散模型的问题

DDPM 的训练目标（预测噪声）是间接的——模型学习的是噪声，而不是直接学习"如何从噪声变成图像"的路径。这导致：
- 需要 1000 步才能生成高质量图像
- 训练目标和推理目标不完全一致

### 6.7.2 Flow Matching 的核心思想

**Flow Matching（Lipman et al., 2022；Liu et al., 2022）** 的思路更直接：**直接学习从噪声到数据的速度场（velocity field）**。

把生成过程看作一个**流（flow）**：

$$\frac{dx_t}{dt} = v_\theta(x_t, t)$$

其中 $v_\theta$ 是模型学习的速度场，$t \in [0, 1]$（$t=0$ 是噪声，$t=1$ 是数据）。

**训练目标**：直接回归速度场

$$\mathcal{L}_{FM} = \mathbb{E}_{t, x_0, x_1}\left[\| v_\theta(x_t, t) - (x_1 - x_0) \|^2\right]$$

其中 $x_t = (1-t) x_0 + t x_1$（线性插值），$x_0 \sim \mathcal{N}(0,I)$（噪声），$x_1 \sim p_{data}$（真实数据）。

```python
def flow_matching_train_step(model, x1):
    """Flow Matching 训练步骤"""
    batch_size = x1.shape[0]
    
    # 采样时间步 t ∈ [0, 1]
    t = torch.rand(batch_size)
    
    # 采样噪声（起点）
    x0 = torch.randn_like(x1)
    
    # 线性插值：xt = (1-t)*x0 + t*x1
    t_expand = t.view(-1, *([1] * (x1.dim() - 1)))
    xt = (1 - t_expand) * x0 + t_expand * x1
    
    # 目标速度：从 x0 到 x1 的方向
    target_velocity = x1 - x0
    
    # 模型预测速度
    pred_velocity = model(xt, t)
    
    # 损失：预测速度 vs 目标速度
    loss = F.mse_loss(pred_velocity, target_velocity)
    return loss

def flow_matching_sample(model, shape, steps=50):
    """Flow Matching 采样（ODE 求解）"""
    x = torch.randn(shape)  # 从噪声出发
    dt = 1.0 / steps
    
    for i in range(steps):
        t = torch.full((shape[0],), i / steps)
        v = model(x, t)  # 预测速度
        x = x + v * dt   # 欧拉积分
    
    return x
```

### 6.7.3 Flow Matching vs DDPM

| 维度 | DDPM | Flow Matching |
|------|------|--------------|
| 训练目标 | 预测噪声 $\epsilon$ | 预测速度 $v = x_1 - x_0$ |
| 路径 | 随机（马尔可夫链） | 直线（最优传输） |
| 推理步数 | 50-1000 步 | **4-50 步** |
| 训练稳定性 | 好 | 更好 |
| 代表模型 | SD 1.x, SDXL | **SD3, FLUX.1, Wan2.1** |

Flow Matching 的路径是**直线**（最优传输），而 DDPM 的路径是弯曲的随机游走。直线路径意味着更少的步数就能到达目标，这是 FLUX.1-schnell 只需 **4 步**就能生成高质量图像的根本原因。

## 6.8 DiT：扩散 Transformer 架构

### 6.8.1 从 UNet 到 DiT

传统扩散模型（SD 1.x/SDXL）使用 **UNet** 作为去噪网络：CNN 下采样 + Transformer 中间层 + CNN 上采样。

**DiT（Diffusion Transformer，Peebles & Xie, 2023）** 用**纯 Transformer** 替代 UNet：

```
输入噪声 [B, C, H, W]
    ↓ Patchify（图像切块）
图像 Patch Token [B, N, D]（N = H/p × W/p）
    ↓ + 位置编码
    ↓ DiT Block × L
    ↓ Unpatchify
输出噪声预测 [B, C, H, W]
```

### 6.8.2 DiT Block：adaLN-Zero

DiT 的核心是 **adaLN-Zero（自适应层归一化）**——用时间步 $t$ 和条件 $c$ 动态调制归一化参数：

```python
class DiTBlock(nn.Module):
    """DiT 核心块：adaLN-Zero + Self-Attention + FFN"""
    def __init__(self, d_model, n_heads, mlp_ratio=4.0):
        super().__init__()
        self.norm1 = nn.LayerNorm(d_model, elementwise_affine=False)
        self.norm2 = nn.LayerNorm(d_model, elementwise_affine=False)
        self.attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ffn = nn.Sequential(
            nn.Linear(d_model, int(d_model * mlp_ratio)),
            nn.GELU(),
            nn.Linear(int(d_model * mlp_ratio), d_model)
        )
        # adaLN-Zero：从条件向量预测 6 个调制参数
        # [shift_attn, scale_attn, gate_attn, shift_ffn, scale_ffn, gate_ffn]
        self.adaLN_modulation = nn.Sequential(
            nn.SiLU(),
            nn.Linear(d_model, 6 * d_model)
        )
        # 初始化为零（Zero 初始化）：训练初期等同于恒等映射
        nn.init.zeros_(self.adaLN_modulation[-1].weight)
        nn.init.zeros_(self.adaLN_modulation[-1].bias)
    
    def forward(self, x, c):
        """
        x: 图像 patch token [B, N, D]
        c: 条件向量（时间步 t + 文本 embedding 融合）[B, D]
        """
        # 从条件向量预测调制参数
        shift_a, scale_a, gate_a, shift_f, scale_f, gate_f = \
            self.adaLN_modulation(c).chunk(6, dim=-1)
        
        # Self-Attention（带 adaLN 调制）
        x_norm = modulate(self.norm1(x), shift_a, scale_a)
        attn_out, _ = self.attn(x_norm, x_norm, x_norm)
        x = x + gate_a.unsqueeze(1) * attn_out  # gate 控制残差强度
        
        # FFN（带 adaLN 调制）
        x_norm = modulate(self.norm2(x), shift_f, scale_f)
        x = x + gate_f.unsqueeze(1) * self.ffn(x_norm)
        
        return x

def modulate(x, shift, scale):
    """adaLN 调制：x * (1 + scale) + shift"""
    return x * (1 + scale.unsqueeze(1)) + shift.unsqueeze(1)
```

### 6.8.3 MM-DiT：FLUX.1 的双流架构

FLUX.1（Black Forest Labs, 2024）在 DiT 基础上提出 **MM-DiT（Multi-Modal DiT）**：图像 token 和文本 token 在**各自独立的流**中处理，通过 Joint Attention 交互。

```python
class MMDiTBlock(nn.Module):
    """FLUX.1 双流 DiT 块"""
    def __init__(self, d_model, n_heads):
        super().__init__()
        # 图像流
        self.img_norm1 = nn.LayerNorm(d_model, elementwise_affine=False)
        self.img_norm2 = nn.LayerNorm(d_model, elementwise_affine=False)
        self.img_ffn = FeedForward(d_model)
        self.img_adaLN = nn.Linear(d_model, 6 * d_model)
        
        # 文本流（独立权重）
        self.txt_norm1 = nn.LayerNorm(d_model, elementwise_affine=False)
        self.txt_norm2 = nn.LayerNorm(d_model, elementwise_affine=False)
        self.txt_ffn = FeedForward(d_model)
        self.txt_adaLN = nn.Linear(d_model, 6 * d_model)
        
        # 联合注意力（图像+文本 token 一起做注意力）
        self.joint_attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
    
    def forward(self, img_tokens, txt_tokens, c):
        """
        img_tokens: 图像 patch token [B, N_img, D]
        txt_tokens: 文本 token [B, N_txt, D]
        c: 时间步条件 [B, D]
        """
        # 各自 adaLN 调制
        img_mod = self.img_adaLN(c)
        txt_mod = self.txt_adaLN(c)
        
        # 联合注意力：图像和文本 token 拼接后一起做注意力
        img_normed = modulate(self.img_norm1(img_tokens), *img_mod[:3].chunk(3, -1))
        txt_normed = modulate(self.txt_norm1(txt_tokens), *txt_mod[:3].chunk(3, -1))
        
        # 拼接：[img_tokens, txt_tokens]
        joint = torch.cat([img_normed, txt_normed], dim=1)
        attn_out, _ = self.joint_attn(joint, joint, joint)
        
        # 分离结果
        img_attn = attn_out[:, :img_tokens.shape[1]]
        txt_attn = attn_out[:, img_tokens.shape[1]:]
        
        # 各自残差 + FFN
        img_tokens = img_tokens + img_attn
        img_tokens = img_tokens + self.img_ffn(modulate(self.img_norm2(img_tokens), *img_mod[3:].chunk(3, -1)))
        
        txt_tokens = txt_tokens + txt_attn
        txt_tokens = txt_tokens + self.txt_ffn(modulate(self.txt_norm2(txt_tokens), *txt_mod[3:].chunk(3, -1)))
        
        return img_tokens, txt_tokens
```

**MM-DiT vs 标准 DiT**：

| 维度 | 标准 DiT（SD3） | MM-DiT（FLUX.1） |
|------|--------------|----------------|
| 文本处理 | 通过交叉注意力注入 | 独立流，联合注意力 |
| 文本理解深度 | 中等 | 更深（文本有自己的 Self-Attention） |
| 参数量 | 较少 | 较多（双流独立权重） |
| 图像质量 | 好 | 更好（FLUX.1 超越 SD3） |

## 6.9 视频生成：时序维度的扩展

### 6.9.1 从图像到视频的核心挑战

视频生成在图像生成基础上增加了**时序维度**，带来三大挑战：

| 挑战 | 描述 | 解决方案 |
|------|------|---------|
| **时序一致性** | 相邻帧物体/场景要连贯 | 时间注意力 / 3D 因果 VAE |
| **物理合理性** | 运动轨迹、光影要符合物理 | 大规模视频预训练 |
| **计算量** | 内存和计算随时长线性增长 | 时空压缩 VAE + 分块推理 |

### 6.9.2 3D 因果 VAE：视频的压缩基础

视频生成的 VAE 需要同时压缩**空间**和**时间**维度：

```python
class CausalVideoVAE(nn.Module):
    """3D 因果 VAE（Wan2.1 / CogVideoX 风格）"""
    def __init__(self, spatial_compress=8, temporal_compress=4):
        super().__init__()
        # 因果卷积：每帧只能看到当前帧和之前的帧（保证时序因果性）
        self.encoder = CausalConv3D(
            in_channels=3,
            out_channels=16,
            spatial_compress=spatial_compress,    # 空间压缩 8×
            temporal_compress=temporal_compress   # 时间压缩 4×
        )
        self.decoder = CausalConv3DDecoder(...)
    
    def encode(self, video):
        """
        video: [B, T, C, H, W]（批次, 帧数, 通道, 高, 宽）
        → latent: [B, T//4, 16, H//8, W//8]
        """
        return self.encoder(video)
    
    def decode(self, latent):
        """潜变量 → 视频帧"""
        return self.decoder(latent)
```

**因果卷积的关键**：每帧的潜变量只依赖于当前帧和之前的帧，不依赖未来帧。这保证了视频生成时可以**流式解码**（逐帧生成），而不需要等待所有帧生成完毕。

### 6.9.3 时空注意力：Video DiT 的核心

视频 DiT 在图像 DiT 基础上增加了**时间维度的注意力**：

```python
class VideoSelfAttention(nn.Module):
    """时空联合注意力（Sora/Kling 风格）"""
    def __init__(self, d_model, n_heads, attn_type="full_3d"):
        super().__init__()
        self.attn_type = attn_type
        
        if attn_type == "full_3d":
            # 3D Full Attention：时间+空间 token 全部一起做注意力
            # 优点：建模能力最强；缺点：计算量 O((T*H*W)²)
            self.attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        
        elif attn_type == "factorized":
            # 分解注意力：先空间，再时间（计算量更小）
            self.spatial_attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
            self.temporal_attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
    
    def forward(self, x):
        """x: [B, T, H*W, D]"""
        B, T, N, D = x.shape
        
        if self.attn_type == "full_3d":
            # 展平时空维度：[B, T*N, D]
            x_flat = x.reshape(B, T * N, D)
            x_flat, _ = self.attn(x_flat, x_flat, x_flat)
            return x_flat.reshape(B, T, N, D)
        
        elif self.attn_type == "factorized":
            # 空间注意力：每帧内部
            x_s = x.reshape(B * T, N, D)
            x_s, _ = self.spatial_attn(x_s, x_s, x_s)
            x = x + x_s.reshape(B, T, N, D)
            
            # 时间注意力：跨帧同位置
            x_t = x.permute(0, 2, 1, 3).reshape(B * N, T, D)
            x_t, _ = self.temporal_attn(x_t, x_t, x_t)
            x = x + x_t.reshape(B, N, T, D).permute(0, 2, 1, 3)
            
            return x
```

### 6.9.4 主流视频生成模型架构对比

| 模型 | VAE | 注意力类型 | 文本编码器 | 训练目标 | 特色 |
|------|-----|-----------|-----------|---------|------|
| **Sora** | 时空 VAE | 3D Full Attention | T5-XXL | Flow Matching | 物理一致性最强 |
| **Kling 2.0** | 3D Causal VAE | 3D + Temporal Causal | T5-XXL + CLIP | Flow Matching | 运动流畅度最优 |
| **Wan2.1** | Wan-VAE（4×8×8） | 3D Full Attention | umt5-xxl（双语） | Flow Matching | 开源 VBench SOTA |
| **Veo 3** | 时空 VAE | Video+Audio 双流 | Gemini 编码器 | Flow Matching | 音视频同步生成 |
| **CogVideoX** | 3D Causal VAE | Expert Transformer | T5-XXL | Flow Matching | 开源，中文支持 |

## 6.10 完整生成流程：从文本到图像

以 Stable Diffusion（LDM + CFG）为例，完整推理流程：

```python
def text_to_image(
    prompt: str,
    negative_prompt: str = "",
    guidance_scale: float = 7.5,
    num_steps: int = 50,
    seed: int = 42
):
    """文生图完整流程（Stable Diffusion 风格）"""
    
    # ── Step 1: 文本编码 ──────────────────────────────
    # 有条件 embedding（正向提示词）
    text_ids = tokenizer(prompt, max_length=77, padding="max_length")
    text_emb = text_encoder(text_ids).last_hidden_state  # [1, 77, 768]
    
    # 无条件 embedding（负向提示词，用于 CFG）
    uncond_ids = tokenizer(negative_prompt, max_length=77, padding="max_length")
    uncond_emb = text_encoder(uncond_ids).last_hidden_state  # [1, 77, 768]
    
    # 拼接（批量处理有条件+无条件）
    text_embeddings = torch.cat([uncond_emb, text_emb])  # [2, 77, 768]
    
    # ── Step 2: 初始化潜变量（纯噪声）────────────────
    torch.manual_seed(seed)
    latents = torch.randn(1, 4, 64, 64)  # [B, C, H/8, W/8]
    latents = latents * scheduler.init_noise_sigma  # 缩放到调度器期望的范围
    
    # ── Step 3: 去噪循环（DDIM 50步）────────────────
    scheduler.set_timesteps(num_steps)
    
    for t in scheduler.timesteps:
        # 复制 latents（有条件+无条件各一份）
        latent_model_input = torch.cat([latents] * 2)
        latent_model_input = scheduler.scale_model_input(latent_model_input, t)
        
        # UNet/DiT 预测噪声（同时处理有条件和无条件）
        with torch.no_grad():
            noise_pred = unet(
                latent_model_input,
                t,
                encoder_hidden_states=text_embeddings
            ).sample
        
        # CFG：分离有条件和无条件预测，放大条件信号
        noise_uncond, noise_cond = noise_pred.chunk(2)
        noise_pred = noise_uncond + guidance_scale * (noise_cond - noise_uncond)
        
        # 调度器更新：计算 x_{t-1}
        latents = scheduler.step(noise_pred, t, latents).prev_sample
    
    # ── Step 4: VAE 解码（潜变量 → 像素）────────────
    latents = latents / 0.18215  # 反缩放
    with torch.no_grad():
        image = vae.decode(latents).sample
    
    # 后处理：[-1,1] → [0,255]
    image = (image / 2 + 0.5).clamp(0, 1)
    image = (image * 255).round().to(torch.uint8)
    
    return image
```

## 6.11 小结：生成模型技术全景

```
技术演进与核心贡献：

AE（2006）
  └─ 核心：编码器-解码器，重建损失
  └─ 问题：潜空间不连续，无法生成新样本

VAE（2013）
  └─ 核心：概率潜空间 + ELBO + 重参数化技巧
  └─ 贡献：可生成新样本，潜空间连续
  └─ 问题：生成图像模糊（MSE 损失）

GAN（2014）
  └─ 核心：生成器 vs 判别器对抗训练
  └─ 贡献：生成质量大幅提升
  └─ 问题：模式崩溃、训练不稳定

DDPM（2020）
  └─ 核心：逐步加噪 + 学习逆转去噪
  └─ 贡献：训练稳定，生成质量超越 GAN
  └─ 问题：推理需要 1000 步，速度慢

LDM（2022）
  └─ 核心：在 VAE 潜空间做扩散
  └─ 贡献：计算量降低 64×，Stable Diffusion 诞生
  └─ 关键：VAE 作为高质量压缩器

CFG（2022）
  └─ 核心：有条件 - 无条件双路推理，放大条件信号
  └─ 贡献：文生图质量飞跃，无需额外分类器
  └─ 公式：ε̃ = ε_uncond + w*(ε_cond - ε_uncond)

Flow Matching（2022）
  └─ 核心：直接学习速度场，直线路径
  └─ 贡献：4步推理，训练更稳定
  └─ 代表：SD3, FLUX.1, Wan2.1

DiT（2023）
  └─ 核心：纯 Transformer 替代 UNet
  └─ 贡献：任意分辨率，扩展性更好
  └─ 变体：MM-DiT（FLUX.1 双流）

Video DiT（2024-2025）
  └─ 核心：3D 因果 VAE + 时空注意力
  └─ 代表：Sora, Kling 2.0, Wan2.1, Veo 3
```

| 技术 | 核心公式/机制 | 代表模型 |
|------|-------------|---------|
| **VAE** | ELBO = 重建 - β·KL | Stable Diffusion 的 VAE |
| **DDPM** | $\mathcal{L} = \|\epsilon - \epsilon_\theta(x_t,t)\|^2$ | DDPM, DDIM |
| **LDM** | 在 $z = E(x)$ 上做扩散 | Stable Diffusion 全系列 |
| **CFG** | $\tilde{\epsilon} = \epsilon_{unc} + w(\epsilon_c - \epsilon_{unc})$ | 所有条件扩散模型 |
| **Flow Matching** | $\mathcal{L} = \|v_\theta - (x_1-x_0)\|^2$ | SD3, FLUX.1, Wan2.1 |
| **DiT** | adaLN-Zero + Patch Attention | SD3, FLUX.1, Sora |
| **MM-DiT** | 双流 Joint Attention | FLUX.1 |
