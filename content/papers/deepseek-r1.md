# 论文解读：DeepSeek-R1

> 不需要蒸馏、不需要人工标注思考链，纯 RL 训练涌现出超越 o1 的推理能力

## 一句话总结

DeepSeek-R1 通过 **GRPO (Group Relative Policy Optimization)** 算法，直接在 DeepSeek-V3 基座上进行强化学习训练，无需蒸馏或人工标注 CoT 数据，让模型自发涌现出长链推理能力，在 AIME 数学竞赛上超越 OpenAI o1。

## 核心问题：为什么需要 R1？

OpenAI o1 展示了「推理时间扩展」的巨大潜力，但它的训练方法高度不透明。外界猜测 o1 依赖：
- 大量人工标注的 CoT 数据
- 从更强模型蒸馏的推理轨迹
- 复杂的多阶段 RLHF 训练

DeepSeek-R1 的核心贡献是证明：**这些都不需要。纯 RL 就够了。**

## 核心方法：GRPO

### 传统 RLHF 的问题

```
传统 RLHF 流程:
Prompt → 模型生成响应 → 奖励模型打分 → PPO 优化策略

问题：
1. 奖励模型本身会出错（reward hacking）
2. 训练奖励模型需要大量人工标注偏好数据
3. PPO 训练不稳定，超参敏感
```

### GRPO 的解决方案

```
GRPO 流程:
Prompt → 模型生成 N 个响应（组采样）→ 外部验证器打分 → 组内排名 → 策略优化

优势：
1. 无需奖励模型（直接用数学求解器/代码执行器验证）
2. 无需人工标注
3. 组内相对排名更稳定
```

关键公式：

```
对于每个 prompt x，采样 G 个响应 {y₁, ..., y_G}
对每个响应 yᵢ，获取外部验证奖励 rᵢ
组内标准化：r̂ᵢ = (rᵢ - mean(r)) / std(r)
策略梯度：∇J = Σᵢ r̂ᵢ · ∇log π(yᵢ|x)
```

### 为什么「组内相对排名」这么重要？

| 方案 | 信号来源 | 稳定性 | 数据需求 |
|------|---------|--------|---------|
| RLHF | 奖励模型（绝对分数） | 低（reward hacking） | 高（人工标注） |
| DPO | 偏好对（相对排序） | 中 | 中（偏好数据） |
| **GRPO** | **组内排名（相对排序）** | **高** | **低（自动验证）** |

关键洞察：对于有明确正确答案的任务（数学、代码），可以完全绕过人工标注，用**外部验证器**（数学求解器、代码执行沙箱）直接判断对错。

## 训练过程中的涌现现象

### R1-Zero：纯 RL 的「裸训练」实验

DeepSeek 先做了一个极端实验：**R1-Zero**——直接在 V3 基座上做 GRPO，不加任何 SFT 数据。

令人惊讶的结果：
1. **自发出现 CoT**：模型学会了「让我一步步思考」的推理模式
2. **自发出现自我验证**：模型学会了「让我检查一下上面的计算」
3. **自发出现多种解题策略**：模型会尝试不同方法并比较

```
R1-Zero 的思考链示例（AIME 数学题）：

<think>
题目要求...让我先分析条件...
方法一：直接代入...
等等，这个计算好像有误，让我用另一种方法验证...
方法二：构造辅助函数...
两种方法得到相同结果，答案应该是...
</think>

答案：42
```

### 但 R1-Zero 有严重缺陷

- **可读性差**：思考链混乱，中英文混杂
- **重复问题**：有时陷入循环推理
- **格式不稳定**：答案提取困难

### R1 的完整训练流程

```
Step 1: 冷启动 SFT
  - 用少量高质量 CoT 数据做 SFT
  - 目的：教会模型基本的推理格式
  - 数据量：仅 ~1000 条

Step 2: GRPO 强化学习（核心阶段）
  - 任务：数学、代码、逻辑推理
  - 验证器：数学求解器 + 代码执行沙箱
  - 训练轮次：~1000 步
  - 关键：推理能力在此阶段涌现

Step 3: 蒸馏回通用能力
  - 将 Step 2 产生的长 CoT 推理能力
  - 蒸馏回 V3 基座，保持通用对话能力

Step 4: 最终 GRPO 精调
  - 加入有用性、安全性奖励
  - 平衡推理能力和对话体验
```

## 关键结论

### 1. RL 可以涌现推理能力

这是本文最重要的发现。不需要人工标注 CoT、不需要蒸馏，纯 RL 就能让模型学会推理。这说明推理能力是 LLM 的「潜在能力」，RL 只是一个释放它的工具。

### 2. 数据效率极高

R1 的 SFT 冷启动仅用 ~1000 条数据。对比 InstructGPT 的数万条人工标注，效率提升了一个数量级。

### 3. 「熵崩塌」是核心挑战

训练过程中最大的风险是熵崩塌——模型的输出多样性急剧下降，陷入固定模式。字节跳动后续提出的 DAPO 方法专门解决了这个问题。

### 4. 开源对齐研究的里程碑

R1 的代码和训练细节完全公开，让整个社区可以复现和改进。这是闭源 o1 做不到的。

## 实际影响

| 维度 | 影响 |
|------|------|
| 学术 | 证明 RL 可以涌现推理能力，挑战了「推理需要蒸馏」的共识 |
| 工程 | GRPO 比 PPO 更简单更稳定，降低了 RL 训练的门槛 |
| 产业 | 开源推理模型的性能首次追上闭源（AIME 超越 o1） |
| 安全 | 推理 Scaling 让模型更「深思熟虑」，但也可能产生更复杂的欺骗行为 |
