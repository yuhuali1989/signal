## 二、网络通信：分布式训练的命脉

### 通信模式与开销

| 并行策略 | 通信操作 | 通信量 | 频率 |
|---------|---------|--------|------|
| 数据并行 (DP) | All-Reduce (梯度) | O(模型大小) | 每步 1 次 |
| 张量并行 (TP) | All-Reduce | O(batch × hidden) | 每层 2 次 |
| 流水线并行 (PP) | P2P Send/Recv | O(batch × hidden) | 每个 micro-batch |
| MoE Expert Parallel | All-to-All | O(batch × expert) | 每个 MoE 层 |

### 通信优化策略

```
1. 计算-通信重叠 (Overlap)
   → 在 GPU 计算当前层时，同步传输上一层的梯度

2. 梯度压缩
   → FP32 → BF16 → FP8 梯度通信，减少 2-4x 带宽

3. 分层 All-Reduce
   → 先节点内 NVLink All-Reduce，再节点间 IB All-Reduce

4. DeepSeek DualPipe
   → 将 Forward 和 Backward 交错执行，隐藏通信延迟
```

## 三、故障恢复：训练的最大敌人

万卡集群中，**GPU 故障是常态而非异常**：

```
故障率估算 (10K GPU 集群):
  单 GPU MTBF (平均无故障时间): ~10,000 小时
  10K GPU 集群 MTBF: ~1 小时
  → 平均每小时就有 1 个 GPU 可能故障！

应对策略:
  1. 频繁 Checkpoint (每 10-30 分钟)
  2. 异步 Checkpoint (不阻塞训练)
  3. 弹性训练 (节点故障自动缩减，恢复后自动扩展)
  4. 热备节点 (预留 5-10% 备用 GPU)
  5. NVIDIA NeMo 的 Fault Tolerance 框架
```
