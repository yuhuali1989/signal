## 一句话总结

NVIDIA 开源的数据飞轮参考架构，通过**自动化数据收集→质量评估→模型蒸馏→部署验证**的闭环流程，持续将大模型（Teacher）的能力蒸馏到小模型（Student），实现 AI Agent 的成本-延迟-精度自动优化。

## 核心问题

企业部署 AI Agent 面临的矛盾：
- **大模型精度高但成本贵**（GPT-5.4: $10/M token, 延迟 2-5s）
- **小模型便宜快速但精度不够**（本地 7B: 免费, 延迟 100ms, 但精度差 20%）
- 手动蒸馏是一次性的，模型不会随数据持续改进

## 关键技术细节

### LLM-as-Judge 质量评分

数据飞轮的核心在于**自动化质量评估**：用一个 LLM 评估另一个 LLM 的输出质量。

```python
# 质量评分 prompt
JUDGE_PROMPT = """评估以下 AI Agent 响应的质量 (1-5分):

用户请求: {input}
Agent 响应: {output}

评分维度:
1. 准确性: 信息是否正确
2. 完整性: 是否回答了所有问题
3. 格式: 结构是否清晰
4. 有用性: 是否真正帮助了用户

输出 JSON: {"score": N, "reason": "..."}
"""
```

### 增量蒸馏策略

- **Base Round**: 用 ~10K 高质量样本做初始 SFT
- **Incremental Round**: 每周新增 ~1K 样本做 LoRA 增量微调
- **DPO Round**: 收集失败案例做 DPO 偏好对齐
- **自动回退**: 如果精度下降 > 2%，自动回退到上一版本
