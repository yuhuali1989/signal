## 一句话总结

本综述首次系统梳理了世界模型在自动驾驶中的应用，将现有工作分为**场景生成、闭环仿真、策略学习、预训练**四大范式，覆盖 80+ 篇论文，并提出了从"数据驱动"到"世界模型驱动"的范式转移路线图。

## 范式 1：场景生成——数据放大器

### 核心思路

```
问题: 自动驾驶数据收集成本极高
  - 正常驾驶数据: $10/公里 (车辆+人员+存储)
  - 长尾场景数据: $1000+/场景 (稀有，难以复现)
  - 极端工况: 几乎无法获取 (事故、恶劣天气)

解决方案: 用世界模型生成合成数据
  真实数据 10K 场景 → 训练世界模型 → 生成 1M+ 场景
  成本: 从 $10M 降到 $100K (100x 降低)
```

### 代表性工作

| 模型 | 机构 | 方法 | 控制维度 |
|------|------|------|---------|
| **GAIA-1** | Wayve | Autoregressive Video | 路线+动作+语言 |
| **MagicDrive** | NTU | Diffusion + BEV Control | 3D bbox+地图+文本 |
| **DriveDreamer** | Li Auto | Latent Diffusion | 高精地图+交通流 |
| **Panacea** | Tsinghua | Panoramic Video Gen | 全景+BEV+LiDAR |

### GAIA-1 架构解析

```
GAIA-1 (Wayve, 2023):

  输入: 视频帧序列 + 驾驶动作 (转向/加速) + 语言描述
    ↓
  Video Tokenizer (VQ-VAE)
  → 将视频编码为离散 token 序列
    ↓
  World Model Transformer (自回归)
  → 根据历史 token + 动作 → 预测下一帧 token
    ↓
  Video Decoder
  → token → 渲染真实感视频
  
  关键能力:
  1. 改变驾驶意图: "左转" → 生成左转场景
  2. 改变天气: "下雨" → 添加雨天效果
  3. 添加车辆: "前方出现一辆卡车" → 合成卡车
  4. 保持物理一致性: 阴影/反射/碰撞物理正确
```

## 范式 2：闭环仿真——虚拟路测

### 与传统仿真的区别

```
传统仿真 (CARLA/SUMO):
  ┌─ 手工建模 3D 资产 (高成本) ─┐
  │ 手工编写车辆行为规则       │ → 渲染质量: ★★☆
  │ 手工设定交通流参数         │ → 物理精度: ★★★
  └──────────────────────────┘ → 多样性: ★☆☆

世界模型仿真 (UniSim/GenAD):
  ┌─ 从真实数据学习世界模型 ──┐
  │ 自动学习物理规律         │ → 渲染质量: ★★★★★
  │ 自动学习交通行为分布     │ → 物理精度: ★★★★
  └──────────────────────────┘ → 多样性: ★★★★★
```

### UniSim 架构

```
UniSim (NVIDIA, 2024-2026):

  Step 1: 神经场景重建
  真实驾驶数据 → 3D Gaussian Splatting → 可编辑 3D 场景
  
  Step 2: 动态仿真
  ┌─── 场景编辑 ───────────────────┐
  │ 移动/添加/删除车辆             │
  │ 改变天气/光照                  │
  │ 修改道路拓扑                   │
  └────────────────────────────────┘
           ↓
  ┌─── 闭环仿真 ───────────────────┐
  │ Agent 策略 → 动作 → 世界模型    │
  │ 世界模型 → 下一帧 → Agent 观察  │
  │ 循环执行 → 完整驾驶场景        │
  └────────────────────────────────┘
  
  Step 3: 评估
  收集碰撞/违规/舒适度指标
```

## 范式 3：策略学习——"白日梦"训练

### MILE: Model-based Imitation Learning

```
MILE 的"白日梦"训练:

  Phase 1: 学习世界模型
  真实数据: (s_t, a_t, s_{t+1}) → 训练 f(s_t, a_t) = s_{t+1}
  
  Phase 2: 在世界模型中训练策略
  ┌─── 想象轨迹 ───────────────────┐
  │ s_0 (真实) → π(s_0)=a_0        │
  │ f(s_0, a_0) = ŝ_1 (想象)       │
  │ π(ŝ_1) = a_1                   │
  │ f(ŝ_1, a_1) = ŝ_2 (想象)       │
  │ ...                             │
  │ 评估想象轨迹的质量(碰撞/效率)    │
  │ 反向传播更新策略 π               │
  └─────────────────────────────────┘
  
  优势:
  1. 无需真实环境交互（安全）
  2. 可以想象极端场景（长尾覆盖）
  3. 策略更新速度 100x（无需等渲染）
  
  限制:
  1. 世界模型的误差会累积（compounding error）
  2. 想象轨迹长度有限（>10 步后严重偏离）
```

### DriveWorld-VLA 的世界模型预训练

```
DriveWorld-VLA 在预训练阶段利用世界模型:

  Stage 1: 世界模型预训练
  ┌─── 自监督任务 ─────────────────┐
  │ 输入: 视频帧 t-2, t-1, t       │
  │ 任务 1: 预测帧 t+1, t+2        │
  │ 任务 2: 预测深度图              │
  │ 任务 3: 预测场景流              │
  │ 无需标注！从视频中自监督学习     │
  └─────────────────────────────────┘
  
  Stage 2: VLA 微调
  预训练的世界模型 → 添加动作解码器 → 端到端微调
  
  效果:
  - 预训练让模型理解物理世界规律
  - 微调只需少量标注数据
  - nuScenes 规划 L2 误差: 1.15m (SOTA)
```

## 关键技术挑战

### 1. 物理一致性

```
常见物理不一致问题:

❌ 车辆穿透: 两车重叠 → 世界模型未学到碰撞
❌ 悬浮物体: 车辆离地面浮起
❌ 阴影错误: 光源方向改变但阴影没变
❌ 车道约束: 车辆行驶在人行道上

解决方案:
1. 物理先验注入: 碰撞检测作为约束条件
2. 可微分物理引擎: 梯度穿过物理引擎回传
3. 对抗训练: 判别器检测物理不一致
4. 结构化表征: 显式建模物体位置和运动状态
```

### 2. 长程预测的累积误差

```
世界模型预测误差随步数指数增长:

  步数:    1      5      10     20     50
  误差:    0.1m   0.8m   3.2m   12.5m  >>
  可用性:  ✅     ✅     ⚠️     ❌     ❌

缓解策略:
1. 潜在空间预测（DriveWorld-VLA）: 减少信息损失
2. 多步训练: 用 teacher forcing + scheduled sampling
3. 分层预测: 高层长期规划 + 低层短期控制
4. 世界模型集成: 多个世界模型投票减少方差
```

### 3. Sim-to-Real 迁移

```
仿真 vs 真实的 Domain Gap:

  视觉差异: 渲染 vs 真实照片
  动力学差异: 仿真物理 vs 真实物理
  传感器差异: 理想传感器 vs 噪声传感器
  分布差异: 仿真的交通流 vs 真实交通流

2026 年 Sim2Real 最佳实践:
1. Domain Randomization: 随机化纹理/光照/天气
2. 神经渲染: 3DGS 生成真实感图像（消除视觉差距）
3. 参数化物理: 在仿真中随机化摩擦系数/质量/延迟
4. 渐进式迁移: 仿真预训练 → 少量真实数据微调
```
