## 5.1 多模态 LLM 的架构范式

多模态 LLM 的核心问题是：如何将视觉/音频等非文本信息编码为 LLM 能理解的 token 序列？

### 三种主流架构

```
方案 A: 编码器 + 投影层 + LLM（LLaVA 系列）
  Image → ViT Encoder → MLP Projector → LLM

方案 B: 原生多模态（GPT-4o / Gemini / Gemma 4）
  Image/Audio/Text → 统一 Tokenizer → 单一 Transformer

方案 C: 交叉注意力（Flamingo / Llama 3.2 / Llama 4）
  Image → ViT → Cross-Attention 注入到 LLM 层间
```

| 架构 | 代表模型 | 优点 | 缺点 |
|------|---------|------|------|
| 编码器+投影 | LLaVA, InternVL | 实现简单，可复用预训练 ViT | 模态融合不够深 |
| 原生多模态 | GPT-4o, Gemini, Gemma 4 | 统一架构，多模态生成 | 训练成本极高 |
| 交叉注意力 | Flamingo, Llama 3.2/4 | 融合更深，效果好 | 架构复杂度高 |

### 架构选型决策树

```python
def choose_multimodal_arch(budget, performance_req, modalities):
    """多模态架构选型指南"""
    if budget == "unlimited" and "generation" in performance_req:
        return "原生多模态"  # GPT-4o/Gemini: 最强但最贵
    
    if len(modalities) == 2 and "text" in modalities:
        if "vision" in modalities:
            return "编码器+投影"  # LLaVA: 最快上手
        if "audio" in modalities:
            return "编码器+适配器"  # Whisper + LLM
    
    if performance_req == "sota":
        return "交叉注意力"  # Flamingo/Llama 4: 最佳融合
    
    return "编码器+投影"  # 默认方案
```

## 5.2 视觉编码器：从 ViT 到 SigLIP

视觉 Transformer（ViT）将图像切成 patch 后作为序列输入：

$$x = [x_{cls}; \; x_{p_1} E; \; x_{p_2} E; \; \dots; \; x_{p_N} E] + E_{pos}$$

其中 patch 大小通常为 $14 \times 14$ 或 $16 \times 16$，一张 $336 \times 336$ 的图像产生 576 个视觉 token。

### 视觉编码器演进

| 编码器 | 年份 | 特点 | 代表模型 |
|--------|:---:|------|---------|
| ViT-L | 2020 | 标准 ViT，ImageNet 预训练 | 早期 LLaVA |
| CLIP ViT | 2021 | 文本-图像对比学习，语义对齐好 | LLaVA-1.5 |
| SigLIP | 2023 | Sigmoid loss 替代 Softmax，不需要全局 batch | InternVL2, PaLI |
| DINOv2 | 2023 | 自监督学习，细粒度特征强 | 分割/检测任务 |
| NaViT | 2024 | 原生可变分辨率，无需 resize | Gemini 1.5+ |
| AIMv2 | 2025 | Apple 自研，端侧优化 | Apple Intelligence |

### SigLIP vs CLIP 的关键差异

```python
# CLIP: Softmax 对比损失（需要全局 batch 同步）
def clip_loss(image_emb, text_emb, temperature):
    # 需要 all_gather 收集全局 batch
    logits = image_emb @ text_emb.T / temperature  # [B, B]
    labels = torch.arange(B)
    loss = (F.cross_entropy(logits, labels) + 
            F.cross_entropy(logits.T, labels)) / 2
    return loss

# SigLIP: Sigmoid 二元损失（单 GPU 即可）
def siglip_loss(image_emb, text_emb, temperature, bias):
    logits = image_emb @ text_emb.T * temperature + bias  # [B, B]
    labels = 2 * torch.eye(B) - 1  # +1 for pairs, -1 for non-pairs
    loss = -F.logsigmoid(labels * logits).mean()
    return loss
```

SigLIP 的优势：
1. **无需全局 batch**：每个 GPU 独立计算，训练效率提升 2-3x
2. **更好的零样本迁移**：在 27 个评测上平均超越 CLIP
3. **灵活的分辨率**：配合 NaViT 支持任意输入尺寸

### 动态分辨率（LLaVA-NeXT / InternVL2）

支持任意分辨率输入，通过切分 + 缩略图组合：

```
高分辨率图片 (1344×896)
  → 切分为 2×3 = 6 个子图 (448×448 each)
  → 每个子图 ViT 编码 → 576 tokens
  → 总计 6 × 576 + 576(缩略图) = 4032 visual tokens
```

### 视觉 Token 压缩技术

4032 个视觉 token 会严重消耗上下文窗口。主流压缩方案：

| 方案 | 压缩比 | 方法 | 代表 |
|------|:---:|------|------|
| Perceiver Resampler | 4-16x | 可学习查询 + 交叉注意力 | Flamingo, Qwen-VL |
| Spatial Pooling | 2-4x | 平均池化/卷积降采样 | LLaVA-1.6 |
| Token Merging (ToMe) | 2-4x | 相似 token 合并 | InternVL2 |
| Q-Former | 8-16x | 查询 Transformer | BLIP-2 |

## 5.3 视觉-语言对齐训练

## 5.4 语音模态：从 Whisper 到 GPT-4o

### 语音理解

语音 → 特征提取（Mel-spectrogram）→ 语音编码器（Whisper-style）→ LLM

```python
class SpeechUnderstanding:
    """语音理解 pipeline"""
    
    def __init__(self):
        self.feature_extractor = MelSpectrogram(
            sample_rate=16000,
            n_fft=400,
            n_mels=80,
            hop_length=160
        )
        self.encoder = WhisperEncoder()  # Transformer encoder
        self.projector = nn.Linear(1280, 4096)  # 映射到 LLM 维度
    
    def encode(self, audio_waveform):
        # 30 秒音频 → 80x3000 Mel 特征 → 1500 个语音 token
        mel = self.feature_extractor(audio_waveform)
        speech_features = self.encoder(mel)
        speech_tokens = self.projector(speech_features)
        return speech_tokens
```

### 语音生成（Speech Synthesis）

GPT-4o 实现了**端到端语音生成**，无需 TTS 管线：

$$\text{Text/Audio Input} \rightarrow \text{Unified Model} \rightarrow \text{Audio Tokens} \rightarrow \text{Vocoder} \rightarrow \text{Waveform}$$

关键突破：GPT-4o 能实时理解说话人的语气、情绪，并以适当的语调回应，延迟 < 300ms。

```
DeepSeek-R1 代码推理过程:
<think>
1. 这个问题需要实现二叉树的最大路径和
2. 路径可以不经过根节点
3. 使用递归：每个节点计算经过它的最大路径
4. 全局变量维护最大值
5. 注意负数节点的处理
</think>

def maxPathSum(self, root):
    self.max_sum = float('-inf')
    def dfs(node):
        if not node: return 0
        left = max(dfs(node.left), 0)   # 负数就不选
        right = max(dfs(node.right), 0)
        self.max_sum = max(self.max_sum, left + right + node.val)
        return max(left, right) + node.val
    dfs(root)
    return self.max_sum
```

## 5.7 前沿：统一多模态生成

下一代多模态模型的趋势是**理解和生成统一**：

### 统一生成架构

```python
class UnifiedMultimodalModel:
    """理解+生成统一架构（Janus-Pro 风格）"""
    
    def __init__(self):
        # 理解编码器（已有模态→token）
        self.vision_encoder = SigLIP()
        self.audio_encoder = USM()
        
        # 生成解码器（token→新模态）
        self.image_decoder = DiffusionHead()
        self.audio_decoder = VocoderHead()
        
        # 统一 Transformer 骨干
        self.backbone = TransformerDecoder(
            hidden_dim=4096,
            num_layers=32,
            num_heads=32
        )
    
    def forward(self, inputs, task):
        # 编码所有输入模态
        tokens = self.encode_all_modalities(inputs)
        
        # Transformer 处理
        hidden = self.backbone(tokens)
        
        # 根据任务选择输出模态
        if task == "text_generation":
            return self.text_head(hidden)
        elif task == "image_generation":
            return self.image_decoder(hidden)
        elif task == "speech_synthesis":
            return self.audio_decoder(hidden)
```
