## 5.1 多模态 LLM 的架构范式

多模态 LLM 的核心问题是：如何将视觉/音频等非文本信息编码为 LLM 能理解的 token 序列？

### 三种主流架构

```
方案 A: 编码器 + 投影层 + LLM（LLaVA 系列）
  Image → ViT Encoder → MLP Projector → LLM

方案 B: 原生多模态（GPT-4o / Gemini / Gemma 4）
  Image/Audio/Text → 统一 Tokenizer → 单一 Transformer

方案 C: 交叉注意力（Flamingo / Llama 3.2 / Llama 4）
  Image → ViT → Cross-Attention 注入到 LLM 层间
```

| 架构 | 代表模型 | 优点 | 缺点 |
|------|---------|------|------|
| 编码器+投影 | LLaVA, InternVL | 实现简单，可复用预训练 ViT | 模态融合不够深 |
| 原生多模态 | GPT-4o, Gemini, Gemma 4 | 统一架构，多模态生成 | 训练成本极高 |
| 交叉注意力 | Flamingo, Llama 3.2/4 | 融合更深，效果好 | 架构复杂度高 |

### 架构选型决策树

```python
def choose_multimodal_arch(budget, performance_req, modalities):
    """多模态架构选型指南"""
    if budget == "unlimited" and "generation" in performance_req:
        return "原生多模态"  # GPT-4o/Gemini: 最强但最贵
    
    if len(modalities) == 2 and "text" in modalities:
        if "vision" in modalities:
            return "编码器+投影"  # LLaVA: 最快上手
        if "audio" in modalities:
            return "编码器+适配器"  # Whisper + LLM
    
    if performance_req == "sota":
        return "交叉注意力"  # Flamingo/Llama 4: 最佳融合
    
    return "编码器+投影"  # 默认方案
```

### 5.1.5 Embedding 入门：文本、图像、音频的向量化对比

在进入具体架构前，许多读者会有一个朴素但关键的疑问：**文本 Embedding 是"查词表"得到向量（例如词表大小 65536，每个词分配一个 ID），那图像和音频既没有"词表"、也没有天然的"词"，它们的 Embedding 到底是怎么来的？**

这一节用一条主线把三种模态串起来：**所有 Embedding 的本质都是"把原始信号映射成一个 $d$ 维向量（例如 $d=4096$），只是获取向量的路径不同"**。

#### 1. 文本 Embedding：离散 ID → 查表

文本的 Embedding 有一个明显的"离散"特性，所以流程最直接：

```
原文: "你好"
  ↓ Tokenizer (BPE / SentencePiece)
Token ID: [872, 1234]        # 词表大小 V=65536
  ↓ nn.Embedding(V, d) 查表
向量:     [[0.12, -0.33, ..., 0.51],   # 4096 维
          [0.44,  0.02, ..., -0.08]]
```

关键点：
- 词表大小 $V$ 是固定的（例如 65536），Embedding 矩阵形状是 $V \times d$。
- 每个 token ID 对应矩阵的一行；"查表"就是 `embedding_table[id]`。
- 这个 Embedding 矩阵的参数是**训练出来的**，反向传播会更新每个 ID 对应的行。

```python
import torch.nn as nn
text_embed = nn.Embedding(num_embeddings=65536, embedding_dim=4096)
ids = torch.tensor([872, 1234])
x = text_embed(ids)           # shape: [2, 4096]
```

#### 2. 图像 Embedding：连续像素 → Patch → 线性投影

**图像没有天然的词表**（因为像素是连续值、而且数量爆炸：一张 $224 \times 224$ 的图有 50176 个像素，不可能像文本那样枚举一个有限词表）。所以图像 Embedding 采用完全不同的思路——**"切块 + 投影"**：

```
原图: [3, 224, 224]              # 3 通道，224×224 像素
  ↓ 切成 16×16 的 patch
Patches: 14×14 = 196 个 patch，每个 [3, 16, 16] = 768 个像素值
  ↓ Flatten 成 768 维向量
Vectors: [196, 768]
  ↓ 线性投影 nn.Linear(768, d)
Image Embedding: [196, 4096]     # 和文本 embedding 同维度 d=4096
```

关键对比：

| 方面 | 文本 | 图像 |
|------|------|------|
| 基本单位 | token（有限词表 65536） | patch（无限连续像素块） |
| 获取向量 | 查表（离散索引 → 行） | 线性投影（连续向量 → 连续向量） |
| 一条输入 | 几十~几千个 token | 固定 196/576 个 patch token |
| Embedding 矩阵 | $V \times d$ | $(P \times P \times 3) \times d$ |

**一句话**：图像没有 ID 这回事，它是用 `nn.Linear` 把 patch 的像素向量直接"压缩"成 $d$ 维向量；词表的等价物是"所有可能的 patch 组合"，但我们不枚举它，而是学一个线性函数来处理任意 patch。

```python
class PatchEmbed(nn.Module):
    def __init__(self, patch_size=16, in_ch=3, d=4096):
        super().__init__()
        # 用一个 Conv2d 同时完成"切 patch + 线性投影"
        self.proj = nn.Conv2d(in_ch, d, kernel_size=patch_size, stride=patch_size)
    def forward(self, img):             # img: [B, 3, 224, 224]
        x = self.proj(img)              # [B, d, 14, 14]
        x = x.flatten(2).transpose(1,2) # [B, 196, d]
        return x
```

这正是 **ViT（Vision Transformer）** 的第一步，后面 5.2 节会展开。

#### 3. 音频 Embedding：波形 → 频谱 → 两条路线

音频比图像还多一个"时间"维度。**原始音频是一维波形**（例如 16kHz 采样 1 秒钟 = 16000 个浮点数），太长且冗余，所以先要做"压缩表示"：

```
原始波形: [16000] 个浮点采样值（1 秒）
  ↓ STFT + Mel 滤波器组
Mel 频谱图: [80, 100]             # 80 个频带 × 100 个时间帧
                                   ← 到这里它就变成"一张图"了
```

接下来有**两条主流路线**，区别在于要不要离散化：

##### 路线 A：连续特征路线（Whisper / Qwen2-Audio，偏理解）

和图像思路几乎一样，把 Mel 频谱当成"图"来切 patch：

```
Mel 频谱 [80, 100]
  ↓ Conv1d / Patch 切块
Audio patches [T, 768]
  ↓ 线性投影
Audio Embedding [T, 4096]
```

特点：embedding 是**连续向量**，能喂给 LLM 做理解，但模型**无法直接生成音频**（因为输出是连续向量，难以还原波形）。

##### 路线 B：离散 Codebook 路线（Encodec / SoundStream，GPT-4o 采用，支持生成）

这条路线才是真正能让"音频有词表"的方案：用一个叫 **VQ-VAE（Vector Quantization VAE）** 的模型，把音频压缩成一串离散整数 ID，**像文本 token 一样**！

```
波形 → Encoder → 连续特征 z [T, 128]
                       ↓
                 在 Codebook（大小 K=1024）中找最近邻
                       ↓
              离散 token ID [T]  ← 每帧映射到 1024 个"音频码"之一
                       ↓
              nn.Embedding(1024, d) 查表
                       ↓
              Audio Embedding [T, 4096]
```

这样音频就"伪装"成了文本：原来的词表大小是 65536 个词，现在扩展为"65536 个词 + 1024 个音频码"，LLM 用**完全相同**的下一 token 预测机制同时处理文字和声音。GPT-4o 能在一个模型里又说话又唱歌，就是因为这条路线。

**三种 Embedding 对比总览**：

```mermaid
flowchart TB
    subgraph 文本
        T1[字符串 你好] --> T2[Tokenizer]
        T2 --> T3[Token ID: 872, 1234<br/>词表 V=65536]
        T3 --> T4[查表 nn.Embedding]
        T4 --> T5[向量 2, 4096]
    end
    subgraph 图像
        I1[像素张量 3, 224, 224] --> I2[切 patch 16x16]
        I2 --> I3[196 个 patch, 每个 768 维]
        I3 --> I4[线性投影 nn.Linear]
        I4 --> I5[向量 196, 4096]
    end
    subgraph 音频离散路线
        A1[波形 16000] --> A2[Mel 频谱 80, 100]
        A2 --> A3[VQ-VAE 编码 + 最近邻]
        A3 --> A4[Codebook ID, 1024 类]
        A4 --> A5[查表 nn.Embedding]
        A5 --> A6[向量 T, 4096]
    end
```

#### 4. 为什么最后 $d$ 都一样？

你可能已经注意到：三种 Embedding 最后都输出 $d=4096$（或 LLM 设定的任何维度）。这不是巧合——**它们必须维度一样，才能拼在同一条序列里送进 Transformer**：

```
输入序列 = [文本 token₁, 文本 token₂, 图像 patch₁, ..., 图像 patch₁₉₆, 音频码₁, ...]
         每个元素都是 4096 维向量
         ↓
      同一个 Transformer
```

这也是为什么 5.1 节所有架构（LLaVA / Flamingo / GPT-4o）本质都是"想办法把各模态变成 $d$ 维 token 序列"——**维度对齐是多模态融合的底线约定**。

> 记住这张总对比表，后续所有 ViT、SigLIP、Whisper、VQ-VAE 的细节都是在填充它的某一行。

| 模态 | 原始形态 | 中间表示 | 获取向量方式 | 是否有"词表" |
|------|---------|---------|------------|------------|
| 文本 | 字符串 | Token ID | 查 Embedding 表 | 有（BPE 词表 ~65K） |
| 图像 | 像素矩阵 | Patch 向量 | 线性投影 | 无（连续投影） |
| 音频（连续路线） | 波形 | Mel 频谱 Patch | 线性投影 | 无 |
| 音频（离散路线） | 波形 | Codebook ID | VQ-VAE 量化 + 查表 | 有（学出来的 ~1K 码） |

铺垫完成，下面正式进入视觉编码器的工程细节。

## 5.2 视觉编码器：从 ViT 到 SigLIP

视觉 Transformer（ViT）将图像切成 patch 后作为序列输入：

$$x = [x_{cls}; \; x_{p_1} E; \; x_{p_2} E; \; \dots; \; x_{p_N} E] + E_{pos}$$

其中 patch 大小通常为 $14 \times 14$ 或 $16 \times 16$，一张 $336 \times 336$ 的图像产生 576 个视觉 token。

### 视觉编码器演进

| 编码器 | 年份 | 特点 | 代表模型 |
|--------|:---:|------|---------|
| ViT-L | 2020 | 标准 ViT，ImageNet 预训练 | 早期 LLaVA |
| CLIP ViT | 2021 | 文本-图像对比学习，语义对齐好 | LLaVA-1.5 |
| SigLIP | 2023 | Sigmoid loss 替代 Softmax，不需要全局 batch | InternVL2, PaLI |
| DINOv2 | 2023 | 自监督学习，细粒度特征强 | 分割/检测任务 |
| NaViT | 2024 | 原生可变分辨率，无需 resize | Gemini 1.5+ |
| AIMv2 | 2025 | Apple 自研，端侧优化 | Apple Intelligence |

### SigLIP vs CLIP 的关键差异

```python
# CLIP: Softmax 对比损失（需要全局 batch 同步）
def clip_loss(image_emb, text_emb, temperature):
    # 需要 all_gather 收集全局 batch
    logits = image_emb @ text_emb.T / temperature  # [B, B]
    labels = torch.arange(B)
    loss = (F.cross_entropy(logits, labels) + 
            F.cross_entropy(logits.T, labels)) / 2
    return loss

# SigLIP: Sigmoid 二元损失（单 GPU 即可）
def siglip_loss(image_emb, text_emb, temperature, bias):
    logits = image_emb @ text_emb.T * temperature + bias  # [B, B]
    labels = 2 * torch.eye(B) - 1  # +1 for pairs, -1 for non-pairs
    loss = -F.logsigmoid(labels * logits).mean()
    return loss
```

SigLIP 的优势：
1. **无需全局 batch**：每个 GPU 独立计算，训练效率提升 2-3x
2. **更好的零样本迁移**：在 27 个评测上平均超越 CLIP
3. **灵活的分辨率**：配合 NaViT 支持任意输入尺寸

### 动态分辨率（LLaVA-NeXT / InternVL2）

支持任意分辨率输入，通过切分 + 缩略图组合：

```
高分辨率图片 (1344×896)
  → 切分为 2×3 = 6 个子图 (448×448 each)
  → 每个子图 ViT 编码 → 576 tokens
  → 总计 6 × 576 + 576(缩略图) = 4032 visual tokens
```

### 视觉 Token 压缩技术

4032 个视觉 token 会严重消耗上下文窗口。主流压缩方案：

| 方案 | 压缩比 | 方法 | 代表 |
|------|:---:|------|------|
| Perceiver Resampler | 4-16x | 可学习查询 + 交叉注意力 | Flamingo, Qwen-VL |
| Spatial Pooling | 2-4x | 平均池化/卷积降采样 | LLaVA-1.6 |
| Token Merging (ToMe) | 2-4x | 相似 token 合并 | InternVL2 |
| Q-Former | 8-16x | 查询 Transformer | BLIP-2 |

## 5.3 视觉-语言对齐训练

### 三阶段训练策略（2026 最佳实践）

**Stage 1: 预训练投影层**
- 冻结 ViT 和 LLM，只训练投影层
- 数据：图文配对数据（如 LAION-5B）
- 目标：让投影层学会将视觉特征映射到语言空间
- 通常 1 epoch，数小时完成

**Stage 2: 高质量对齐微调**
- 解冻 LLM（或用 LoRA），冻结或解冻 ViT
- 数据：多模态指令数据（VQA、图表理解、OCR 等）
- 目标：赋予模型遵循多模态指令的能力
- 数据量：1-5M 样本

**Stage 3: 对齐与安全（RLHF/DPO for Vision）**
- 多模态偏好数据
- 减少幻觉（Hallucination）：模型描述图中不存在的内容
- 提升细粒度理解：空间关系、计数、文字识别

### 多模态幻觉问题

多模态模型的一个核心挑战是 **视觉幻觉**：

```python
# 典型幻觉场景
prompt = "这张图片中有几只猫？"
image = "一张有 2 只猫的图片"

# 幻觉回答: "图片中有 3 只猫，一只橘猫在沙发上..."
# 正确回答: "图片中有 2 只猫"

# 缓解方法
class HallucinationMitigation:
    def __init__(self):
        self.strategies = [
            "对比解码: 减少语言先验的影响",
            "RLHF-V: 基于细粒度标注的偏好学习",
            "视觉 Grounding: 回答与图像区域绑定",
            "自一致性: 多次采样+投票验证",
        ]
```

## 5.4 语音模态：从 Whisper 到 GPT-4o

### 语音理解

语音 → 特征提取（Mel-spectrogram）→ 语音编码器（Whisper-style）→ LLM

```python
class SpeechUnderstanding:
    """语音理解 pipeline"""
    
    def __init__(self):
        self.feature_extractor = MelSpectrogram(
            sample_rate=16000,
            n_fft=400,
            n_mels=80,
            hop_length=160
        )
        self.encoder = WhisperEncoder()  # Transformer encoder
        self.projector = nn.Linear(1280, 4096)  # 映射到 LLM 维度
    
    def encode(self, audio_waveform):
        # 30 秒音频 → 80x3000 Mel 特征 → 1500 个语音 token
        mel = self.feature_extractor(audio_waveform)
        speech_features = self.encoder(mel)
        speech_tokens = self.projector(speech_features)
        return speech_tokens
```

### 语音生成（Speech Synthesis）

GPT-4o 实现了**端到端语音生成**，无需 TTS 管线：

$$\text{Text/Audio Input} \rightarrow \text{Unified Model} \rightarrow \text{Audio Tokens} \rightarrow \text{Vocoder} \rightarrow \text{Waveform}$$

关键突破：GPT-4o 能实时理解说话人的语气、情绪，并以适当的语调回应，延迟 < 300ms。

### 语音 Token 化方案对比

| 方案 | 编码器 | Token 率 | 特点 |
|------|--------|:---:|------|
| Whisper-style | Transformer | 50 tok/s | 语义丰富，语音理解强 |
| SpeechTokenizer | RVQ (残差量化) | 50-75 tok/s | 保留声学细节，可生成 |
| Encodec (Meta) | Neural Codec | 24-75 tok/s | 高压缩比，用于 MusicGen |
| USM (Google) | Conformer | 25 tok/s | 支持 300+ 语言 |
| VoxCPM2 (2026) | Tokenizer-free | N/A | 跳过离散化，连续表征直接输入 |

### 端到端语音对话的架构演进

```
2023: ASR → LLM → TTS（级联，延迟 2-5s）
2024: GPT-4o 统一模型（端到端，延迟 300ms）
2025: Moshi/SpiritLM 开源全双工（可打断，延迟 200ms）
2026: VoxCPM2 无 Tokenizer（连续语音表征，延迟 150ms）
```

### 5.5.2 FIM（Fill-in-the-Middle）训练

```
原始代码:  def add(a, b): return a + b
FIM 格式:  <PRE>def add(a, b):<SUF>+ b<MID> return a <EOT>
```

FIM 使模型能够在给定上下文和后文的情况下填充中间代码，这是代码补全的核心能力。

### 5.5.3 代码推理（Code Reasoning）

2025-2026 年的新趋势：不只是生成代码，还要**推理代码**：

```
传统代码模型: 输入 → 直接生成代码
代码推理模型: 输入 → 分析问题 → 制定方案 → 生成代码 → 验证

DeepSeek-R1 代码推理过程:
<think>
1. 这个问题需要实现二叉树的最大路径和
2. 路径可以不经过根节点
3. 使用递归：每个节点计算经过它的最大路径
4. 全局变量维护最大值
5. 注意负数节点的处理
</think>

def maxPathSum(self, root):
    self.max_sum = float('-inf')
    def dfs(node):
        if not node: return 0
        left = max(dfs(node.left), 0)   # 负数就不选
        right = max(dfs(node.right), 0)
        self.max_sum = max(self.max_sum, left + right + node.val)
        return max(left, right) + node.val
    dfs(root)
    return self.max_sum
```

## 5.7 多模态理解模型：主流架构深度解析

多模态**理解**模型的核心任务是：给定图像/视频/音频，回答问题、描述内容、推理关系。2026 年已形成三大主流架构流派。

### 5.7.1 流派一：编码器 + 投影层（Projector-based）

这是最主流的工程路线，代表作 LLaVA 系列和 InternVL 系列。

**核心思路**：用预训练好的视觉编码器（ViT/SigLIP）提取视觉特征，通过一个轻量投影层（MLP 或 Q-Former）将视觉 token 映射到 LLM 的词嵌入空间，然后拼接文本 token 一起送入 LLM。

```
┌─────────────────────────────────────────────────────────────┐
│                  Projector-based 架构                        │
│                                                             │
│  Image ──→ [ViT/SigLIP] ──→ Visual Features                │
│                                    │                        │
│                              [MLP Projector]                │
│                                    │                        │
│  Text ──→ [Tokenizer] ──→ Text Tokens                      │
│                                    │                        │
│              [Visual Tokens ⊕ Text Tokens] ──→ [LLM] ──→ 答案│
└─────────────────────────────────────────────────────────────┘
```

**LLaVA 系列演进**：

| 版本 | 投影层 | 视觉编码器 | 关键改进 |
|------|--------|-----------|---------|
| LLaVA-1.0 | 单层 Linear | CLIP ViT-L | 概念验证，仅 595K 对齐数据 |
| LLaVA-1.5 | 2层 MLP | CLIP ViT-L/336px | MLP 替代 Linear，效果大幅提升 |
| LLaVA-NeXT | 2层 MLP | CLIP ViT-L | 动态高分辨率（最高 4 tiles） |
| LLaVA-OneVision | 2层 MLP | SigLIP-SO400M | 单图/多图/视频统一，SOTA |

```python
# LLaVA-1.5 投影层实现（核心就是两层 MLP）
class LLaVAProjector(nn.Module):
    def __init__(self, vision_dim=1024, llm_dim=4096):
        super().__init__()
        self.proj = nn.Sequential(
            nn.Linear(vision_dim, llm_dim),
            nn.GELU(),
            nn.Linear(llm_dim, llm_dim),
        )
    
    def forward(self, visual_features):
        # visual_features: [B, N_patches, vision_dim]
        # 输出: [B, N_patches, llm_dim]，直接拼接到文本 token 前
        return self.proj(visual_features)

# 前向传播：视觉 token 插入文本序列
def llava_forward(image, text_tokens, model):
    # 1. 视觉编码
    visual_feats = model.vision_encoder(image)  # [B, 576, 1024]
    visual_tokens = model.projector(visual_feats)  # [B, 576, 4096]
    
    # 2. 找到 <image> 占位符位置，替换为视觉 token
    img_pos = (text_tokens == IMAGE_TOKEN_ID).nonzero()
    input_embeds = model.embed_tokens(text_tokens)
    input_embeds[img_pos] = visual_tokens.reshape(-1, 4096)
    
    # 3. LLM 自回归生成
    return model.llm(inputs_embeds=input_embeds)
```

**InternVL2 的关键创新**：

InternVL2 是 2025 年开源多模态理解的最强模型之一，核心创新在于：

1. **InternViT-6B**：自研 60 亿参数视觉编码器，远超 CLIP ViT-L（307M）
2. **动态分辨率 + Pixel Shuffle**：将 4×4 的视觉 token 合并为 1 个，压缩 16 倍
3. **渐进式训练**：先对齐视觉编码器，再联合微调

```python
# InternVL2 的 Pixel Shuffle 压缩（4x4 → 1 token）
def pixel_shuffle(visual_tokens, scale_factor=0.5):
    """
    将相邻的 4 个 token 合并为 1 个（空间降采样）
    visual_tokens: [B, H*W, C] → [B, H/2*W/2, C*4]
    """
    B, N, C = visual_tokens.shape
    H = W = int(N ** 0.5)
    x = visual_tokens.reshape(B, H, W, C)
    # 2x2 空间合并
    x = x.reshape(B, H//2, 2, W//2, 2, C)
    x = x.permute(0, 1, 3, 2, 4, 5).reshape(B, H//2 * W//2, C * 4)
    return x  # token 数量减少 4 倍，维度增加 4 倍
```

### 5.7.2 流派二：交叉注意力注入（Cross-Attention Injection）

代表作：Flamingo（DeepMind）、Llama 3.2 Vision、Llama 4。

**核心思路**：不把视觉 token 拼接到文本序列，而是在 LLM 的每隔 N 层插入一个**交叉注意力层**，让文本 token 主动"查询"视觉特征。

```
┌─────────────────────────────────────────────────────────────┐
│                  Cross-Attention 架构（Flamingo）             │
│                                                             │
│  Image ──→ [ViT] ──→ Visual Features (K, V)                │
│                              │                              │
│  Text ──→ [LLM Layer 1] ──→ [LLM Layer 2]                 │
│                                    │                        │
│                         [Cross-Attn: Q=text, K/V=visual]   │
│                                    │                        │
│                         [LLM Layer 3] ──→ ... ──→ 答案      │
└─────────────────────────────────────────────────────────────┘
```

```python
class FlamingoLayer(nn.Module):
    """Flamingo 的门控交叉注意力层"""
    def __init__(self, d_model, n_heads):
        super().__init__()
        self.cross_attn = nn.MultiheadAttention(d_model, n_heads)
        self.tanh_gate = nn.Parameter(torch.zeros(1))  # 初始为 0，不影响原始 LLM
        self.norm = nn.LayerNorm(d_model)
    
    def forward(self, text_hidden, visual_features):
        # text_hidden: [L_text, B, D]（文本 token 的隐状态）
        # visual_features: [L_vis, B, D]（视觉特征）
        
        # 交叉注意力：文本 query 视觉 key/value
        attn_out, _ = self.cross_attn(
            query=text_hidden,
            key=visual_features,
            value=visual_features
        )
        # 门控：tanh(gate) 初始为 0，训练中逐渐开放
        return text_hidden + self.tanh_gate.tanh() * self.norm(attn_out)
```

**Flamingo vs LLaVA 的本质区别**：

| 维度 | LLaVA（投影层） | Flamingo（交叉注意力） |
|------|---------------|---------------------|
| 视觉融合深度 | 浅（只在输入层） | 深（每隔 N 层都融合） |
| 多图支持 | 困难（上下文爆炸） | 原生支持（交叉注意力天然多图） |
| LLM 冻结 | 通常解冻 | 可完全冻结（门控初始为 0） |
| 训练效率 | 高（参数少） | 低（需要额外交叉注意力层） |
| 视频理解 | 需要特殊处理 | 天然支持（时序帧作为多图） |

**Llama 4 的 Early Fusion**：

Llama 4 进一步演进，采用 **Early Fusion**——在 Embedding 层就将视觉 token 和文本 token 统一编码，而非在中间层注入：

```python
# Llama 4 Early Fusion：视觉 token 和文本 token 在同一序列中
class Llama4EarlyFusion(nn.Module):
    def __init__(self):
        self.text_embed = nn.Embedding(vocab_size, d_model)
        self.vision_encoder = SigLIP()
        self.vision_projector = nn.Linear(vision_dim, d_model)
        # 视觉 token 和文本 token 共享同一个 Transformer
        self.transformer = LlamaTransformer()
    
    def forward(self, text_ids, images):
        # 文本 embedding
        text_embeds = self.text_embed(text_ids)  # [B, L_text, D]
        
        # 视觉 embedding（投影到同一维度）
        vis_feats = self.vision_encoder(images)
        vis_embeds = self.vision_projector(vis_feats)  # [B, L_vis, D]
        
        # 统一序列：[vis_tokens, text_tokens]
        combined = torch.cat([vis_embeds, text_embeds], dim=1)
        
        # 单一 Transformer 处理所有模态
        return self.transformer(combined)
```

### 5.7.3 流派三：原生多模态（Native Multimodal）

代表作：GPT-4o、Gemini 1.5/2.0、Gemma 4。

**核心思路**：从预训练阶段就将所有模态的 token 混合在一起训练，没有独立的"视觉编码器"和"投影层"，所有模态共享同一个 Tokenizer 和 Transformer。

```
┌─────────────────────────────────────────────────────────────┐
│                  原生多模态架构（Gemini 风格）                  │
│                                                             │
│  Image ──→ [Patch Tokenizer] ──→ Image Tokens              │
│  Audio ──→ [Spectrogram Tokenizer] ──→ Audio Tokens        │
│  Text  ──→ [SentencePiece] ──→ Text Tokens                 │
│                                                             │
│  [Image Tokens ⊕ Audio Tokens ⊕ Text Tokens]               │
│                        │                                    │
│              [统一 Transformer（所有模态共享权重）]             │
│                        │                                    │
│              [Image/Audio/Text 输出头]                       │
└─────────────────────────────────────────────────────────────┘
```

**Gemini 的多模态 Tokenizer 设计**：

```python
# Gemini 风格的统一 Tokenizer（概念示意）
class GeminiTokenizer:
    def __init__(self):
        self.text_tokenizer = SentencePiece(vocab_size=256_000)
        self.image_tokenizer = PatchTokenizer(patch_size=16, d_model=4096)
        self.audio_tokenizer = SpectrogramTokenizer(hop_ms=20, d_model=4096)
    
    def tokenize(self, inputs):
        tokens = []
        for item in inputs:
            if item.type == "text":
                tokens.extend(self.text_tokenizer.encode(item.content))
            elif item.type == "image":
                # 图像切成 patch，每个 patch 是一个 token
                tokens.extend(self.image_tokenizer.encode(item.content))
            elif item.type == "audio":
                # 音频切成帧，每帧是一个 token
                tokens.extend(self.audio_tokenizer.encode(item.content))
        return tokens  # 所有模态统一为 token 序列
```

**原生多模态的核心优势**：

1. **无模态边界**：模型内部没有"这是视觉 token，那是文本 token"的区分，融合最彻底
2. **多模态生成**：同一模型可以生成文本、图像、音频（理解和生成统一）
3. **长上下文多模态**：Gemini 1.5 Pro 的 10M token 上下文可以包含数小时视频
4. **涌现能力**：模态间的隐式对齐产生意想不到的跨模态推理能力

**代价**：训练成本极高，需要海量多模态数据，工程复杂度远超投影层方案。

### 5.8.2 扩散模型核心原理

扩散模型（Diffusion Model）的核心思想：**学习逆转加噪过程**。

**前向过程（加噪）**：

$$q(x_t | x_{t-1}) = \mathcal{N}(x_t; \sqrt{1-\beta_t} x_{t-1}, \beta_t I)$$

经过 T 步后，原始图像 $x_0$ 变为纯高斯噪声 $x_T \sim \mathcal{N}(0, I)$。

**反向过程（去噪，模型学习的部分）**：

$$p_\theta(x_{t-1} | x_t) = \mathcal{N}(x_{t-1}; \mu_\theta(x_t, t), \Sigma_\theta(x_t, t))$$

模型 $\epsilon_\theta$ 学习预测每一步加入的噪声：

$$\mathcal{L} = \mathbb{E}_{x_0, \epsilon, t} \left[ \| \epsilon - \epsilon_\theta(x_t, t, c) \|^2 \right]$$

其中 $c$ 是条件（文本 embedding），通过**交叉注意力**注入去噪网络。

```python
# 扩散模型推理（DDIM 采样，简化版）
def ddim_sample(model, text_prompt, steps=50, guidance_scale=7.5):
    # 文本编码
    text_emb = clip_encoder(text_prompt)        # [1, 77, 768]
    uncond_emb = clip_encoder("")               # 无条件 embedding
    
    # 从纯噪声开始
    x = torch.randn(1, 4, 64, 64)  # 潜空间噪声（VAE 压缩后）
    
    for t in reversed(range(steps)):
        # Classifier-Free Guidance: 有条件 - 无条件，放大差异
        noise_cond   = model(x, t, text_emb)
        noise_uncond = model(x, t, uncond_emb)
        noise_pred = noise_uncond + guidance_scale * (noise_cond - noise_uncond)
        
        # DDIM 去噪步骤
        x = ddim_step(x, noise_pred, t)
    
    # VAE 解码：潜空间 → 像素空间
    image = vae_decoder(x)
    return image
```

### 5.8.3 UNet vs DiT：去噪网络的架构革命

**UNet 架构（SD 1.x / SDXL）**：

```
输入噪声 [B, 4, H, W]
    │
[Conv 下采样] × 4 → 特征图逐渐缩小
    │
[中间 Transformer 块]（交叉注意力注入文本）
    │
[Conv 上采样] × 4 → 特征图逐渐恢复（Skip Connection）
    │
输出噪声预测 [B, 4, H, W]
```

**DiT 架构（SD3 / FLUX.1 / Sora）**：

DiT（Diffusion Transformer）用纯 Transformer 替代 UNet，将图像 patch 化后作为序列处理：

```python
class DiTBlock(nn.Module):
    """DiT 的核心块：自注意力 + 交叉注意力 + FFN"""
    def __init__(self, d_model, n_heads):
        super().__init__()
        self.self_attn = nn.MultiheadAttention(d_model, n_heads)
        self.cross_attn = nn.MultiheadAttention(d_model, n_heads)  # 注入文本
        self.ffn = FeedForward(d_model)
        # adaLN-Zero：时间步 t 和条件 c 通过 AdaLN 注入
        self.adaLN = AdaptiveLayerNorm(d_model)
    
    def forward(self, x, t_emb, text_emb):
        # x: 图像 patch token [B, N_patches, D]
        # t_emb: 时间步 embedding [B, D]
        # text_emb: 文本 embedding [B, L_text, D]
        
        # AdaLN：用时间步调制归一化参数
        x = self.adaLN(x, t_emb)
        
        # 自注意力（patch 之间互相关注）
        x = x + self.self_attn(x, x, x)[0]
        
        # 交叉注意力（patch 关注文本）
        x = x + self.cross_attn(x, text_emb, text_emb)[0]
        
        # FFN
        x = x + self.ffn(x)
        return x
```

**UNet vs DiT 对比**：

| 维度 | UNet（SDXL） | DiT（SD3/FLUX） |
|------|-------------|----------------|
| 架构 | CNN + Transformer 混合 | 纯 Transformer |
| 分辨率扩展 | 需要重新训练 | 天然支持任意分辨率 |
| 文本理解 | 交叉注意力（中等） | 双流 Transformer（更强） |
| 训练稳定性 | 较好 | 更好（Transformer 成熟） |
| 推理速度 | 快 | 稍慢（参数更多） |
| 图像质量 | 好 | 更好（SD3/FLUX 超越 SDXL） |

### 5.8.4 FLUX.1：当前开源图像生成最强

FLUX.1（Black Forest Labs，2024）是目前开源图像生成的质量标杆：

**核心创新**：

1. **双流 Transformer（MM-DiT）**：文本和图像 token 分别在各自的流中处理，通过交叉注意力交互，而非共享权重

```python
class MMDiTBlock(nn.Module):
    """FLUX.1 的双流 Transformer 块"""
    def __init__(self, d_model, n_heads):
        super().__init__()
        # 图像流
        self.img_self_attn = nn.MultiheadAttention(d_model, n_heads)
        self.img_ffn = FeedForward(d_model)
        # 文本流
        self.txt_self_attn = nn.MultiheadAttention(d_model, n_heads)
        self.txt_ffn = FeedForward(d_model)
        # 双流交叉注意力（图像↔文本）
        self.cross_attn = JointAttention(d_model, n_heads)
    
    def forward(self, img_tokens, txt_tokens, t_emb):
        # 双流联合注意力：图像和文本 token 一起做注意力
        img_tokens, txt_tokens = self.cross_attn(img_tokens, txt_tokens)
        
        # 各自的 FFN
        img_tokens = img_tokens + self.img_ffn(img_tokens)
        txt_tokens = txt_tokens + self.txt_ffn(txt_tokens)
        
        return img_tokens, txt_tokens
```

2. **Flow Matching 替代 DDPM**：更直接的训练目标，收敛更快

$$\mathcal{L}_{FM} = \mathbb{E}_{t, x_0, x_1} \left[ \| v_\theta(x_t, t) - (x_1 - x_0) \|^2 \right]$$

3. **三种变体**：FLUX.1-dev（开源研究）、FLUX.1-schnell（4步快速推理）、FLUX.1-pro（商业最强）

### 5.8.5 视频生成：从 Sora 到 Kling

视频生成在图像生成基础上增加了**时序维度**的建模。

**核心挑战**：
- 时序一致性：相邻帧之间的物体/场景要连贯
- 物理合理性：运动轨迹、光影变化要符合物理规律
- 长视频生成：内存和计算量随时长线性增长

**主流架构**：

```
Video DiT 架构（Sora/Kling/Veo 风格）:

视频帧 → [时空 Patch 化] → 时空 Token 序列
                                    │
                         [3D DiT（时间+空间联合注意力）]
                                    │
                         [文本条件（交叉注意力）]
                                    │
                         [时空 Token 序列] → [解码] → 视频帧
```

```python
# 视频 DiT 的时空注意力（简化）
class VideoSelfAttention(nn.Module):
    """时空联合注意力：同时建模空间和时间关系"""
    def __init__(self, d_model, n_heads):
        super().__init__()
        # 空间注意力：同一帧内的 patch 互相关注
        self.spatial_attn = nn.MultiheadAttention(d_model, n_heads)
        # 时间注意力：不同帧的同一位置 patch 互相关注
        self.temporal_attn = nn.MultiheadAttention(d_model, n_heads)
    
    def forward(self, x):
        # x: [B, T, H*W, D]（B批次, T帧, 空间patch, 维度）
        B, T, N, D = x.shape
        
        # 空间注意力：每帧内部
        x_spatial = x.reshape(B*T, N, D)
        x_spatial = self.spatial_attn(x_spatial, x_spatial, x_spatial)[0]
        x = x + x_spatial.reshape(B, T, N, D)
        
        # 时间注意力：跨帧同位置
        x_temporal = x.permute(0, 2, 1, 3).reshape(B*N, T, D)
        x_temporal = self.temporal_attn(x_temporal, x_temporal, x_temporal)[0]
        x = x + x_temporal.reshape(B, N, T, D).permute(0, 2, 1, 3)
        
        return x
```

### 5.9.1 为什么要统一？

传统方案的问题：

```

生成模型（SD/FLUX）:          只能生成图像，不能理解

```

统一模型的优势：
1. **端到端**：理解和生成在同一模型内完成，无信息损失
2. **互相增强**：生成能力提升理解（通过生成验证理解），理解能力提升生成（更好的语义控制）
3. **更少参数**：一个模型替代两个

### 5.9.2 统一架构的三种路线

**路线 A：自回归统一（AR-based）**

代表：LlamaGen、Chameleon（Meta）、Janus-Pro（DeepSeek）

核心思路：将图像 token 化（VQ-VAE/VQGAN），与文本 token 一起做自回归预测。

```python

    def __init__(self):
        # 统一词表：文本 token + 图像 token
        self.vocab_size = 32000 + 8192  # 文本词表 + 图像码本
        self.embed = nn.Embedding(self.vocab_size, d_model)
        self.transformer = CausalTransformer()
        
        # 图像 tokenizer（VQ-VAE）
        self.image_tokenizer = VQVAE(codebook_size=8192)
    
    def generate_image(self, text_prompt):
        # 文本 token
        text_ids = tokenizer(text_prompt)
        
        # 自回归生成图像 token（256 个，对应 16×16 patch）
        image_ids = []
        for _ in range(256):
            logits = self.transformer(text_ids + image_ids)
            next_token = sample(logits[-1, 32000:])  # 只从图像词表采样
            image_ids.append(next_token + 32000)
        
        # 解码图像 token → 像素
        return self.image_tokenizer.decode(image_ids)
```

class TransfusionModel(nn.Module):

        self.text_head = nn.Linear(d_model, vocab_size)  # AR 文本生成
        self.diffusion_head = nn.Linear(d_model, patch_dim)  # 扩散图像生成
    
    def forward(self, tokens, mode):
        hidden = self.transformer(tokens)
        
        if mode == "text":
            # 自回归：预测下一个文本 token
            return self.text_head(hidden)
        elif mode == "image":
            # 扩散：预测噪声（去噪目标）
            return self.diffusion_head(hidden)
