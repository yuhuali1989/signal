
## 技术细节

### 架构选择

Llama 3 选择 **Dense Transformer** 而非 MoE 的理由：
1. 训练稳定性更好（MoE 的路由不稳定问题）
2. 推理延迟更可预测
3. 开源生态兼容性更好（MoE 需要特殊推理支持）

但这也意味着推理成本更高——405B Dense 的推理成本约为同效果 MoE 的 3-5 倍。

### GQA 注意力

Llama 3 使用 **Grouped Query Attention (GQA)**：

| 模型 | 注意力头 | KV 头 | GQA 比率 |
|------|---------|-------|---------|
| 8B | 32 | 8 | 4:1 |
| 70B | 64 | 8 | 8:1 |
| 405B | 128 | 8 | 16:1 |

GQA 在保持模型质量的同时将 KV Cache 显存降低了 4-16 倍。

### 长上下文扩展

从 8K 扩展到 128K 的渐进训练策略：

```
阶段 1: 8K 上下文，15T tokens
阶段 2: 渐进扩展 8K → 32K → 128K
         通过 RoPE 频率调整（θ = 500,000）
         短序列 + 长序列混合训练
```

## 影响与后续

Llama 3 的影响远超模型本身：

1. **开源生态爆发**：基于 Llama 3 的微调模型数量超过 10,000 个
2. **企业级采用**：大量企业从 GPT-4 API 迁移到自部署 Llama 3
3. **推动竞争**：直接刺激了 Qwen、GLM、Mistral 等开源模型的快速迭代
4. **Scaling Law 验证**：15T tokens + 405B params 的训练验证了 Chinchilla 法则在超大规模的适用性

后续 Llama 4 在此基础上引入了 MoE 架构和原生多模态，延续了开源的主线。
