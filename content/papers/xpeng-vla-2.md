## 1. 研究背景与动机

### 1.1 VLA 模型的量产困境

截至 2025 年底，VLA 模型在学术基准上表现优异（DriveWorld-VLA nuScenes SOTA），但量产落地面临三大挑战：

1. **算力门槛**：训练 VLA 需要万卡级集群，多数车企负担不起
2. **迭代速度**：传统感知-规划分离系统数周迭代一次，端到端 VLA 更慢
3. **泛化性**：学术模型在 nuScenes 上好，中国复杂交通场景下退化严重

### 1.2 小鹏的独特优势

- **数据规模**：500 万+车队，日均采集 PB 级驾驶数据
- **算力合作**：阿里云 3 万卡 GPU 集群专项支持
- **场景覆盖**：中国 300+ 城市的复杂交通数据

## 2. 技术架构

### 2.1 模型架构

```
                    ┌────────────────────────┐
                    │     VLA 2.0 架构        │
                    ├────────────────────────┤
                    │                        │
    图像 ──→ 视觉编码器 ──→┐              │
                         ├──→ 72B 基座 LLM ──→ 脑内推理 ──→ 轨迹规划
    LiDAR ──→ 点云编码器 ─→┘    (MoE)        ↑        │
                                              │        ↓
    地图/导航 ──→ 文本编码 ──────────────────→┘    控制指令
                    │                        │
                    └────────────────────────┘
```

**核心组件**：
- **72B MoE 基座模型**：8 专家 × 9B，活跃参数 ~9B
- **视觉编码器**：BEVFormer v3 变体，8 相机 → 统一 BEV
- **脑内推理模块**：类 CoT 的内部推理，但在 latent space 进行（不生成文本）
- **轨迹解码器**：扩散模型生成多模态轨迹候选

### 2.2 脑内推理（Brain Reasoning）

VLA 2.0 的关键创新是**脑内推理**——在模型内部进行多步推理，但不像 Alpamayo-R1 那样生成显式文本 CoT：

```
传统 VLA:    感知 → 直接输出轨迹
显式 CoT:    感知 → "前方有行人" → "减速避让" → 轨迹
脑内推理:    感知 → [latent step 1] → [latent step 2] → 轨迹
                     ↑ 无文本生成，延迟更低 ↑
```

脑内推理的优势：
- 推理延迟比文本 CoT 低 5-10x
- 仍保留推理带来的安全性提升
- 可解释性通过 attention map 而非文本实现

### 2.3 5 天全链路迭代

```
Day 1: 数据采集 → 自动标注 → 质量过滤
Day 2: 困难场景挖掘 → 数据增强 → 构建训练集
Day 3: 分布式训练（3 万卡，~12 小时全量训练）
Day 4: 仿真验证 → Shadow 模式对比 → 安全检查
Day 5: 灰度发布 → 先锋用户 OTA → 回收反馈数据
       ↓
Day 6 → Day 1（下一轮迭代）
```

对比：
- **Tesla FSD**: 72 小时迭代（但范围更小）
- **Waymo**: ~2 周迭代
- **传统方案**: 4-8 周迭代

## 4. 数据闭环工程

### 4.1 数据飞轮架构

```
┌──────────────────────────────────────────────┐
│              XPeng 数据飞轮                    │
├──────────────────────────────────────────────┤
│                                              │
│  500万+车队 → 日均 PB 级数据                  │
│       ↓                                      │
│  自动标注系统（4D时空重建 + 后验优化）          │
│       ↓                                      │
│  困难场景挖掘（异常检测 + 接管分析）            │
│       ↓                                      │
│  训练集构建（课程学习 + 数据混合）              │
│       ↓                                      │
│  3万卡分布式训练（12h全量/4h增量）              │
│       ↓                                      │
│  仿真+Shadow验证                             │
│       ↓                                      │
│  OTA推送 → 回收更多数据 → 闭环                │
│                                              │
└──────────────────────────────────────────────┘
```

### 4.2 关键数据工程创新

1. **4D 时空自动标注**：利用未来帧回溯重建高精度 3D 标注（类 Tesla 4D Auto-Labeling）
2. **困难场景主动挖掘**：基于 VLA 内部不确定性检测，自动识别模型困惑的场景
3. **合成数据增强**：使用世界模型（类 Cosmos）生成罕见场景的合成训练数据
4. **课程学习策略**：从简单场景开始，逐步引入困难场景，训练效率提升 2x

## 5. 关键洞察

### 5.1 开源 VLA 的战略意义

小鹏开源 VLA 2.0 的动机：
1. **生态建设**：吸引更多开发者和车企加入 VLA 生态
2. **数据网络效应**：更多部署 = 更多数据 = 更好的模型
3. **技术输出**：从智能电车制造商转型为 AI 技术供应商（大众合作）

### 5.2 脑内推理 vs 显式 CoT

脑内推理是一个折中方案：
- 不如显式 CoT 可解释，但延迟远低
- 不如直接输出快，但安全性更好
- 适合量产场景的实际需求

### 5.3 中国 VLA 生态的竞争格局

```
开源路线：小鹏 VLA 2.0 (Apache 2.0) → 大众等合作
闭源路线：华为 ADS 3.0 → 自有品牌 + 赋能车企
混合路线：地平线 SuperDrive 2.0 → 芯片+软件捆绑
```

## 6. 局限与未来

1. **72B 基座仍需大量算力**：虽然活跃参数 9B，但完整训练需要 3 万卡
2. **中国场景偏好**：在欧美场景的泛化性待验证（大众合作将是关键测试）
3. **安全认证**：Apache 2.0 开源但不包含安全认证流程
4. **与 Tesla 的数据差距**：500 万车 vs 700 万车，日均数据量差距约 3x
