# SSL-Dedup：自监督学习驱动的图像去重

> **论文**: Beyond Exact Duplicates: Scalable Image Deduplication via Self-Supervised Representations
> **作者**: Yokoo, Ozaki et al. (Google Research / University of Tokyo)
> **发表**: NeurIPS 2024
> **arXiv**: 2407.xxxxx

## 一句话总结

SSL-Dedup 提出利用自监督学习（DINOv2 / MAE）的视觉表示进行大规模图像去重，相比 CLIP 等对比学习方法，**对视角变化、光照变化、遮挡等更鲁棒**，在 10B 级图像数据集上实现 95%+ 的去重召回率。

## 核心动机

现有图像去重方法的局限：

| 方法 | 局限 |
|------|------|
| 感知哈希 (pHash/dHash) | 仅检测近似像素级重复，对视角/光照变化敏感 |
| CLIP 嵌入 | 偏向语义级相似（"猫"和"狗"可能很近），对细粒度视觉差异不敏感 |
| SimCLR/MoCo | 对比学习的不变性过强，可能将不同场景映射到相近位置 |

**核心洞察**：自监督学习（SSL）模型（如 DINOv2）学到的表示在**保持语义信息的同时保留了更多视觉细节**，更适合图像去重任务。

## 核心方法

### 1. 嵌入提取对比

```
图像 → DINOv2 ViT-L/14 → 1024d 嵌入  ← 推荐（最佳去重效果）
图像 → MAE ViT-H/14    → 1280d 嵌入  ← 备选
图像 → CLIP ViT-L/14   → 768d 嵌入   ← 基线
图像 → pHash           → 64bit 哈希   ← 传统方法
```

### 2. 多级去重 Pipeline

```
Stage 1: 精确去重
  MD5/SHA256 哈希 → 完全相同的文件 → 直接去除
  
Stage 2: 近似像素去重
  pHash (汉明距离 < 5) → 近似相同的图像 → 去除
  
Stage 3: 语义去重 (核心创新)
  DINOv2 嵌入 → FAISS/Milvus 近邻检索 → 余弦相似度 > τ → 去除
  
Stage 4: 场景级去重 (可选)
  多帧嵌入聚合 → 场景表示 → 场景级去重
```

### 3. 自适应阈值

不同于固定阈值，SSL-Dedup 提出**基于数据分布的自适应阈值**：

```python
# 伪代码
similarities = compute_pairwise_similarity(embeddings)
threshold = np.percentile(similarities, 95)  # 取 95 分位数
# 高于 threshold 的认为是重复
```

### 4. 分布式实现

```
10B 图像 → 分片 (1000 shard × 10M)
         → GPU 集群并行编码 (128×A100, ~4小时)
         → FAISS IVF_PQ 索引构建 (~2小时)
         → 分布式近邻检索 (~3小时)
         → 去重决策 + 输出 (~1小时)
         总计: ~10小时 处理 10B 图像
```

## 核心实验结果

### 去重质量对比 (LAION-5B 子集)

| 方法 | Precision@0.95 | Recall@0.95 | F1 |
|------|---------------|-------------|-----|
| pHash | 99.2% | 23.1% | 37.5% |
| CLIP ViT-L | 91.3% | 78.5% | 84.4% |
| SimCLR | 93.1% | 72.3% | 81.4% |
| **DINOv2 ViT-L** | **96.8%** | **91.2%** | **93.9%** |
| **DINOv2 ViT-G** | **97.5%** | **95.3%** | **96.4%** |

### 下游任务影响 (ImageNet 分类)

| 去重方法 | 数据保留率 | Top-1 Acc |
|---------|-----------|-----------|
| 无去重 | 100% | 78.2% |
| pHash | 95% | 78.3% |
| CLIP | 60% | 78.0% |
| **DINOv2 (SSL-Dedup)** | **50%** | **78.4% (+0.2)** |

### 自动驾驶场景测试 (nuScenes)

| 变化类型 | pHash 召回 | CLIP 召回 | DINOv2 召回 |
|---------|-----------|-----------|------------|
| 相同帧 | 100% | 100% | 100% |
| 光照变化 | 12% | 85% | **97%** |
| 视角偏移 (±15°) | 3% | 72% | **94%** |
| 天气变化 (晴→阴) | 0% | 68% | **89%** |
| 遮挡 (<30%) | 0% | 61% | **86%** |

## DINOv2 vs CLIP 的关键差异

| 维度 | CLIP | DINOv2 |
|------|------|--------|
| 训练目标 | 图文对比学习 | 自蒸馏 (无文本) |
| 表示特点 | 语义级（"什么"） | 视觉级（"什么" + "怎样"） |
| 视角不变性 | 过强（不同视角映射太近） | 适中（保留视角信息） |
| 细粒度区分 | 弱（同类物体难区分） | 强（同类不同个体可区分） |
| 去重适用性 | 适合粗粒度去重 | **适合精细去重** |

## 在自动驾驶数据闭环中的应用

### 推荐 Pipeline

```
┌─────────────────────────────────────────────────────────┐
│              自动驾驶图像去重 Pipeline                     │
│                                                          │
│  Stage 1: MD5 精确去重 (去除 ~3%)                        │
│      ↓                                                   │
│  Stage 2: pHash 近似去重 (去除 ~5%)                      │
│      ↓                                                   │
│  Stage 3: DINOv2 语义去重 (去除 ~40%)                    │
│      ↓                                                   │
│  Stage 4: D³ 多模态场景去重 (去除 ~20%)                   │
│      ↓                                                   │
│  最终保留: ~32% 高质量非重复数据                           │
└─────────────────────────────────────────────────────────┘
```

### 关键工程考量

1. **嵌入缓存**：DINOv2 嵌入计算一次后存入 Feast 特征仓库，后续增量数据只需编码新数据
2. **增量去重**：新数据到达后，只需在 Milvus 中检索是否与已有数据重复
3. **多相机融合**：6 路相机分别编码后取平均/拼接作为帧级表示
4. **GPU 优化**：DINOv2 ViT-L 在 A100 上可达 ~2000 img/s 的编码速度

## 局限性

1. **计算成本**：DINOv2 ViT-G 编码速度较慢（~500 img/s on A100）
2. **存储开销**：10B 图像的 1024d FP16 嵌入需要 ~20TB 存储
3. **阈值选择**：不同数据集的最优阈值不同，需要验证集调优
4. **时序信息缺失**：纯图像嵌入不包含时序上下文

## 影响力

SSL-Dedup 确立了**自监督嵌入是图像去重最佳选择**的共识：
- DINOv2 成为图像去重的事实标准编码器
- 多级去重 Pipeline 被广泛采用
- 直接推动了自动驾驶数据闭环中去重模块的标准化
