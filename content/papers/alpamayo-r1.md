## 核心问题

现有 VLA 模型有个致命缺陷：**推理和行动脱节**。

```
典型问题场景：

模型看到前方行人   →  推理输出："检测到行人，应该减速"
                    →  但轨迹解码器输出：维持原速
                    →  结果：险些碰撞

根因：推理(language)和动作(action)由不同模块生成，
     没有因果约束确保两者一致
```

## 核心创新

### 1. Chain of Causation (CoC) — 因果推理链

不是简单的"描述场景→输出动作"，而是建立**因果链**：

```
传统 VLA 推理：
  "前方有行人" → "应该减速" → 输出轨迹

Alpamayo-R1 的因果推理链 (CoC)：
  1. 关键实体识别：前方 15m 处行人正在过马路
  2. 因果分析：行人移动方向与本车轨迹将在 2.3s 后交叉
  3. 风险评估：如不减速，1.8s 后距离 < 2m (危险)
  4. 决策推理：需在 0.5s 内开始制动，目标减速至 15km/h
  5. 轨迹约束：生成的轨迹必须满足 d(t=2.3s) > 3m
  → 输出：减速轨迹 + 完整因果解释
```

CoC 数据集构建流程：

```
真实驾驶场景 → 自动标注器(因果标签) → 人在回路验证 → CoC 训练数据
```
