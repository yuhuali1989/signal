# MAI-1.5 与 2026 多模态开源模型新格局

## 1. 多模态开源的爆发时刻

2026 年 4 月的第二周，三大多模态开源模型几乎同时发布：

| 模型 | 发布方 | 参数量 | 许可证 | 亮点 |
|------|--------|--------|--------|------|
| **MAI-1.5** | 微软 | 72B-MoE / 13B / 3B | Apache 2.0 | 原生 MCP + 工具调用 |
| **GLM-5V-Turbo** | 智谱 AI | ~70B | 开放 | 截图→代码生成 |
| **Gemma 4** | Google DeepMind | 27B/26B-MoE/4B/2B | Apache 2.0 | 端侧部署 E2B |

这不是偶然。2026 年的开源模型竞争已经从"谁的文本能力更强"转向了"谁的多模态更实用"——视觉理解、工具调用、代码生成成为决胜关键。

## 2. MAI-1.5：微软的开源觉醒

### 2.1 架构设计

MAI-1.5 是微软首个以 Apache 2.0 发布的完整基础模型系列。72B MoE 版本采用与 DeepSeek-V3 类似的细粒度专家设计：

```
MAI-1.5-72B-MoE 架构：
├── 共享专家: 2 个 (Always-on)
├── 路由专家: 128 个 (Top-8 激活)
├── 激活参数: ~14B (72B 总参数)
├── 视觉编码器: SigLIP-SO400M + MLP Projector
├── 上下文长度: 128K tokens
└── 特色: 原生 MCP 工具调用协议
```

关键创新点：

1. **原生 MCP 支持**：训练阶段就包含 MCP 协议数据，不需要后处理适配
2. **视觉-代码对齐**：专门的截图→代码训练数据，可以从 UI 截图直接生成前端组件
3. **阶梯式推理**：支持 `think` 标签的推理模式，类似 Claude 的 extended thinking

### 2.3 为什么微软现在开源？

微软过去一直通过 OpenAI 提供 AI 能力。MAI-1.5 的开源标志着战略转向：

- **Azure 需要自有模型**：不能完全依赖 OpenAI，需要降低供应链风险
- **开源社区红利**：Meta (Llama)、Google (Gemma)、智谱 (GLM) 都通过开源获得了巨大的生态优势
- **企业客户需求**：越来越多企业要求可私有部署、可审计的基础模型

## 3. GLM-5V-Turbo：截图即代码

### 3.1 核心能力

GLM-5V-Turbo 的杀手特性是**视觉编码→代码生成的端到端能力**：

```
输入: UI 截图 (任意分辨率)
     ↓
视觉编码器: 动态分辨率切片 + 位置嵌入
     ↓
融合层: 视觉 token + 文本指令 cross-attention
     ↓
代码解码: HTML/CSS/React/Vue 组件
     ↓
输出: 可运行的前端代码
```

实测表现：

| 任务 | GLM-5V-Turbo | GPT-4o | Claude 3.5 Sonnet |
|------|-------------|--------|-------------------|
| 截图→HTML 还原度 | 94.2% | 91.8% | 93.5% |
| 截图→React 组件 | 89.7% | 87.3% | 90.1% |
| OCR 准确率 | 97.3% | 96.1% | 95.8% |
| 中文排版理解 | **98.1%** | 88.2% | 89.7% |

中文排版理解是 GLM-5V-Turbo 的绝对优势——对中文 UI、中文表格、中文公式的识别准确率远超竞品。

### 3.2 技术亮点

1. **动态分辨率编码**：不固定输入尺寸，而是自适应切割为若干 448×448 patch
2. **OCR-Free 文字识别**：不依赖外部 OCR，视觉编码器直接理解文字
3. **布局理解**：不只是识别元素，还理解元素之间的空间关系（flexbox/grid 推断）

## 5. 工具调用：MCP 成为模型标配

2026 年最大的趋势变化之一是：**工具调用从"后处理"变成了"预训练"。**

过去的工具调用流程：
```
预训练模型 → SFT 加入 function calling 格式 → 推理时格式化输出 → 后处理解析
```

现在的流程（MAI-1.5 / Mistral Large 3 Ultra）：
```
预训练数据包含 MCP 协议交互 → 模型原生理解工具语义 → 直接输出 MCP JSON-RPC → 零后处理
```

这意味着：

1. **工具调用准确率提升 30%+**：模型理解工具语义而不只是格式
2. **多步工具编排**：模型可以规划工具调用顺序，处理依赖关系
3. **错误恢复**：工具返回错误时，模型能理解错误语义并重试
