# 第 3 章：指令微调与对齐技术

> 选自《大语言模型从入门到前沿》

## 3.1 从预训练到对齐：为什么需要微调

预训练模型是一个强大的"补全机器"，但它不会：
- 遵循用户指令（它只想续写文本）
- 拒绝有害请求（它不知道什么是"有害"）
- 给出有帮助的回答（它不知道什么是"帮助"）
- 承认自己不知道（它会自信地胡说八道）

**对齐（Alignment）** 的目标：让模型变得 **Helpful（有用）、Honest（诚实）、Harmless（无害）**——3H 原则（Anthropic 提出）。

### 对齐的标准流程

```
预训练模型 (Base Model)
    ↓ Stage 1: SFT（监督微调）
指令微调模型 (SFT Model)
    ↓ Stage 2: RM（奖励模型训练）
奖励模型 (Reward Model)
    ↓ Stage 3: PPO/DPO/GRPO（偏好优化）
对齐后模型 (Aligned Model)
    ↓ Stage 4: Constitutional AI / RLAIF（可选）
安全增强模型 (Safety-enhanced Model)
```

### 对齐的本质是什么

从数学角度，对齐是在**模型能力空间**中找到一个子集——用户期望的行为分布：

$$\pi_{aligned} = \arg\max_{\pi} \; \mathbb{E}_{x \sim \mathcal{D}}[R(x, \pi(x))] - \beta \cdot D_{KL}(\pi \| \pi_{base})$$

即：最大化人类奖励，同时不偏离基础模型太远（保持原有能力）。

## 3.2 SFT：监督微调

### 核心思想

用 `(instruction, response)` 配对数据对模型进行监督微调，损失函数只计算 response 部分：

$$\mathcal{L}_{SFT} = -\sum_{t \in \text{response}} \log P(x_t | x_{<t}; \theta)$$

注意：只在 response token 上计算 loss（instruction token 的 loss 被 mask 掉），这是 SFT 区别于预训练的关键。

### 数据格式（ChatML）

```json
{
  "messages": [
    {"role": "system", "content": "你是一个有用的AI助手。"},
    {"role": "user", "content": "解释什么是 Transformer"},
    {"role": "assistant", "content": "Transformer 是一种基于自注意力..."}
  ]
}
```

### SFT 数据的质量层次

| 数据来源 | 成本 | 质量 | 规模 | 典型用途 |
|----------|:---:|:---:|:---:|---------|
| 人工标注 | 极高 | 最佳 | 1K-50K | 核心能力（数学、代码、安全） |
| GPT-4 蒸馏 | 中 | 高 | 50K-500K | 通用对话、指令跟随 |
| Self-Instruct | 低 | 中 | 100K-1M | 扩充多样性 |
| 自动过滤开源 | 极低 | 参差 | 1M+ | 预热阶段 |

**关键发现（LIMA 论文）**：仅 1000 条精心挑选的 SFT 数据就能让 65B 模型产生高质量对话。**SFT 的本质不是教会模型新知识，而是激活模型已有但"休眠"的能力。**

### LoRA：参数高效微调

全量微调 70B 模型需要 ~500GB 显存。LoRA（Low-Rank Adaptation）的核心思想是**冻结原始权重，只训练低秩增量**：

$$W' = W_0 + \Delta W = W_0 + BA$$

其中 $B \in \mathbb{R}^{d \times r}$，$A \in \mathbb{R}^{r \times d}$，$r \ll d$（通常 $r = 8 \sim 64$）。

```python
class LoRALayer(nn.Module):
    def __init__(self, original_layer, r=16, alpha=32):
        super().__init__()
        self.original = original_layer
        d_out, d_in = original_layer.weight.shape
        self.lora_A = nn.Linear(d_in, r, bias=False)
        self.lora_B = nn.Linear(r, d_out, bias=False)
        self.scaling = alpha / r
        nn.init.zeros_(self.lora_B.weight)  # 初始化为零

    def forward(self, x):
        return self.original(x) + self.lora_B(self.lora_A(x)) * self.scaling
```

### LoRA 变体演进

| 方法 | 年份 | 核心改进 | 可训练参数 | 显存 |
|------|------|---------|----------|------|
| LoRA | 2022 | 低秩矩阵分解 | ~0.1% | ~1.2× |
| QLoRA | 2023 | 4-bit 量化 + LoRA | ~0.1% | **~0.5×** |
| LoRA+ | 2024 | 不同 lr 给 A 和 B | ~0.1% | ~1.2× |
| DoRA | 2024 | 方向+幅度解耦 | ~0.12% | ~1.3× |
| GaLore | 2024 | 梯度低秩投影 | 100% | **~0.6×** |
| rsLoRA | 2025 | Rank-Stabilized 缩放 | ~0.1% | ~1.2× |

## 3.3 RLHF：基于人类反馈的强化学习

### 奖励模型训练

收集人类偏好数据 $(x, y_w, y_l)$（$y_w$ 优于 $y_l$），训练奖励模型：

$$\mathcal{L}_{RM} = -\log \sigma(r(x, y_w) - r(x, y_l))$$

即 Bradley-Terry 偏好模型。奖励模型本质上学习的是**人类偏好的隐式表示**。

### 奖励模型的实践经验

```python
class RewardModel(nn.Module):
    def __init__(self, base_model):
        super().__init__()
        self.backbone = base_model  # 通常用 SFT 模型初始化
        self.reward_head = nn.Linear(hidden_size, 1)
    
    def forward(self, input_ids, attention_mask):
        outputs = self.backbone(input_ids, attention_mask=attention_mask)
        # 取最后一个 token 的 hidden state
        last_hidden = outputs.last_hidden_state[:, -1, :]
        reward = self.reward_head(last_hidden)
        return reward

def rm_loss(rewards_chosen, rewards_rejected, margin=0.0):
    """Bradley-Terry loss with optional margin"""
    return -torch.log(
        torch.sigmoid(rewards_chosen - rewards_rejected - margin)
    ).mean()
```

**关键挑战：奖励模型的泛化能力**

奖励模型的数据来源：
1. 人工标注（最贵但最可靠，通常 50K-200K 对比对）
2. AI 辅助标注（GPT-4 打分，再人工审核）
3. 隐式信号（用户点赞/点踩、重新生成等）

### PPO 优化

有了奖励模型后，使用 PPO（Proximal Policy Optimization）算法优化策略：

$$\mathcal{L}_{PPO} = -\mathbb{E}\left[\min\left(\frac{\pi_\theta(y|x)}{\pi_{\text{old}}(y|x)} A_t, \; \text{clip}\left(\frac{\pi_\theta(y|x)}{\pi_{\text{old}}(y|x)}, 1-\epsilon, 1+\epsilon\right) A_t\right)\right]$$

加上 KL 惩罚防止偏离参考模型过远：

$$\mathcal{L} = \mathcal{L}_{PPO} - \beta \cdot D_{KL}(\pi_\theta \| \pi_{ref})$$

### PPO 训练的四个模型

```
┌─── PPO 训练需要 4 个模型同时在 GPU 上 ───┐
│                                           │
│  1. Policy Model (π_θ)     ← 被优化的模型  │
│  2. Reference Model (π_ref) ← 冻结的 SFT  │
│  3. Reward Model (r)        ← 冻结的 RM   │
│  4. Value Model (V)         ← Critic       │
│                                           │
│  70B PPO 需要: ~1.12 TB 显存               │
│  = 4 × 280GB（每个模型 FP32）              │
│  实际用 FP16/量化: ~280 GB                 │
│  = 4 × A100 80GB（最低配置）               │
└───────────────────────────────────────────┘
```

PPO 的问题：
- 需要同时维护 4 个模型，工程复杂
- 训练不稳定，超参数敏感（lr, KL coeff, clip range）
- 奖励模型可能被 "hacked"（模型学会取悦 RM 而非真正有帮助）

## 3.4 DPO：去掉奖励模型的优雅解法

DPO（Direct Preference Optimization, NeurIPS 2023）的核心洞察：**可以直接从偏好数据优化策略，无需单独训练奖励模型**。

### 数学推导

从 RLHF 的最优策略出发：

$$\pi^*(y|x) = \frac{1}{Z(x)} \pi_{ref}(y|x) \exp\left(\frac{r(x,y)}{\beta}\right)$$

反解出隐式奖励：

$$r(x, y) = \beta \log \frac{\pi_\theta(y|x)}{\pi_{ref}(y|x)} + \beta \log Z(x)$$

代入 Bradley-Terry 模型，$Z(x)$ 项消掉，得到 DPO 损失：

$$\mathcal{L}_{DPO} = -\mathbb{E}\left[\log \sigma\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)\right]$$

### DPO 实现

```python
def dpo_loss(policy_logps_w, policy_logps_l, ref_logps_w, ref_logps_l, beta=0.1):
    """DPO 损失函数 — 极其简洁"""
    # 计算 chosen/rejected 的 log-ratio
    pi_logratios = policy_logps_w - policy_logps_l
    ref_logratios = ref_logps_w - ref_logps_l
    
    # DPO 隐式奖励差
    logits = beta * (pi_logratios - ref_logratios)
    
    # Binary cross-entropy
    loss = -F.logsigmoid(logits).mean()
    
    # 监控指标
    chosen_rewards = beta * (policy_logps_w - ref_logps_w).detach()
    rejected_rewards = beta * (policy_logps_l - ref_logps_l).detach()
    reward_margin = (chosen_rewards - rejected_rewards).mean()
    
    return loss, chosen_rewards.mean(), rejected_rewards.mean(), reward_margin

# DPO 训练只需要 2 个模型：policy + reference
# 比 PPO 省 50% 显存！
```

### DPO vs PPO 对比

| 维度 | PPO (RLHF) | DPO |
|------|:---:|:---:|
| 模型数量 | 4 个 | **2 个** |
| 是否需要 RM | 是 | **否** |
| 训练稳定性 | 低（RL 固有问题） | **高** |
| 超参数 | 多且敏感 | 少（主要是 β） |
| 数据效率 | 需要在线采样 | **离线即可** |
| 上限 | 可能更高（在线探索） | 受限于偏好数据质量 |
| 工程复杂度 | 极高 | **低** |

## 3.5 GRPO：DeepSeek 的强化学习新范式

DeepSeek-R1 使用 GRPO（Group Relative Policy Optimization），去掉了 Critic 模型：

### 核心思想

对同一个 prompt 生成一组（group）回答，用组内相对排名代替绝对奖励：

$$A_i = \frac{r_i - \text{mean}(\mathbf{r})}{\text{std}(\mathbf{r})}$$

$$\mathcal{L}_{GRPO} = -\frac{1}{G}\sum_{i=1}^{G} \min\left(\frac{\pi_\theta}{\pi_{old}} A_i, \; \text{clip}(\cdot) A_i \right) - \beta D_{KL}$$

```python
def grpo_loss(model, ref_model, prompts, group_size=8, beta=0.04):
    """GRPO: Group Relative Policy Optimization"""
    total_loss = 0
    
    for prompt in prompts:
        # 1. 对同一 prompt 生成 G 个回答
        responses = [model.generate(prompt) for _ in range(group_size)]
        
        # 2. 用规则或 RM 计算每个回答的奖励
        rewards = torch.tensor([compute_reward(prompt, r) for r in responses])
        
        # 3. 组内归一化（关键！）
        advantages = (rewards - rewards.mean()) / (rewards.std() + 1e-8)
        
        # 4. PPO-style 策略梯度
        for resp, adv in zip(responses, advantages):
            log_ratio = model.log_prob(resp) - ref_model.log_prob(resp)
            ratio = torch.exp(log_ratio)
            
            clipped_ratio = torch.clamp(ratio, 1 - 0.2, 1 + 0.2)
            loss = -torch.min(ratio * adv, clipped_ratio * adv)
            
            # KL 正则化
            kl = model.log_prob(resp) - ref_model.log_prob(resp)
            loss += beta * kl
            
            total_loss += loss.mean()
    
    return total_loss / len(prompts)
```

### GRPO 的优势

1. **不需要 Critic 模型**：省 50% 显存（相比 PPO）
2. **组内归一化天然抑制 reward hacking**：模型不能通过整体提高分数来取巧
3. **DeepSeek-R1 证明它能从零涌现推理能力**：
   - 无需任何 SFT 数据
   - 纯 RL 训练后出现自发的 CoT（思维链）行为
   - "啊哈时刻（Aha Moment）"：模型自发学会了回溯和自我纠正

### GRPO 在 DeepSeek-R1 中的效果

```
训练阶段:

           → 涌现出 CoT 推理
```

### SimPO：最简方案

SimPO 去掉了 reference model，用**平均 log-likelihood 作为隐式奖励**：

$$r_{SimPO}(x, y) = \frac{1}{|y|} \sum_{t} \log \pi_\theta(y_t | x, y_{<t})$$

$$\mathcal{L}_{SimPO} = -\log \sigma\left(\frac{\beta}{|y_w|}\sum \log\pi(y_w) - \frac{\beta}{|y_l|}\sum \log\pi(y_l) - \gamma\right)$$

只需要 1 个模型，训练极其简单。

## 3.8 对齐的开放问题

### 1. Alignment Tax

对齐通常会降低模型的原始能力（尤其是创造性和极端场景处理）。这被称为 **Alignment Tax**：

```
能力           Base Model    Aligned Model    差距
─────────────────────────────────────────────────
数学推理         95            93             -2%
代码生成         90            88             -2%
创意写作         85            72             -15% ← 明显退化
不当内容拒绝     20            95             +375% ← 大幅提升
```

### 2. 过度拒绝（Over-refusal）

过度对齐导致模型拒绝合理请求：

```
用户: "写一个关于化学反应的教学材料"
过度对齐: "抱歉，我无法提供涉及化学品的内容..."  ← 错误拒绝
正确对齐: "当然！以下是化学反应的基础知识..."
```

### 3. Reward Hacking

模型学会取悦奖励模型的 bug 而非真正变好：
- 回答变得啰嗦（因为 RM 可能偏好长回答）
- 过度使用"列表"和"标题"格式
- 频繁使用对冲语句（"值得注意的是..."）
