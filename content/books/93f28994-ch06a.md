# 第 7 章：3D 生成——从 NeRF 到 Flow-based 3D DiT

> 选自《大语言模型从入门到前沿》

上一章我们系统学习了 2D 生成模型的演进路线（AE → VAE → GAN → DDPM → LDM → DiT）。本章将这条线延伸到**三维空间**——如何从一张图片或一段文本，生成可以在游戏引擎中直接使用的 3D 资产？

```
技术演进路线：
NeRF（2020）→ 3D Gaussian Splatting（2023）→ SDS/DreamFusion（2022）
→ 多视角扩散（Zero-1-to-3, 2023）→ 大规模重建（LRM, 2023）
→ 3D 原生扩散（Shap-E/Point-E, 2023）→ Flow-based 3D DiT（Hunyuan3D, 2025）
→ PBR 材质生成（Hunyuan3D 2.1, 2025）
```

## 7.1 3D 表示方法：先搞清楚"3D"是什么

在讨论 3D 生成之前，必须先理解：**计算机中的 3D 物体到底怎么存储？** 不同的表示方法决定了生成模型的架构设计。

### 7.1.1 五种主流 3D 表示

| 表示方法 | 核心思想 | 优势 | 劣势 | 代表 |
|----------|----------|------|------|------|
| **Mesh（网格）** | 顶点 + 三角面片 | 工业标准，可直接渲染 | 拓扑结构复杂，难以直接生成 | OBJ/GLB 格式 |
| **Point Cloud（点云）** | 空间中的离散点集 | 简单灵活 | 无表面信息，渲染质量差 | Point-E |
| **Voxel（体素）** | 3D 像素网格 | 规则结构，易于卷积 | 内存爆炸（$O(N^3)$） | 3D U-Net |
| **隐式场（SDF/NeRF）** | 用神经网络表示 3D | 连续、分辨率无关 | 推理慢，难以编辑 | NeRF, DeepSDF |
| **3D Gaussian Splatting** | 各向异性高斯椭球 | 实时渲染，训练快 | 内存占用大 | 3DGS |

```
Mesh（三角面片）          Point Cloud（点云）        Implicit Field（隐式场）
    /\                    . . . .                   f(x,y,z) → SDF值
   /  \                  . . . . .                  ┌─────────────┐
  /____\                . . . . . .                 │  MLP 网络    │
 /\    /\              . . . . . . .                │  输入: (x,y,z)│
/  \  /  \            . . . . . . . .               │  输出: σ, c   │
                                                    └─────────────┘
```

### 7.1.2 Mesh 为什么是最终目标？

无论中间用什么表示，**最终产出必须是 Mesh**，因为：

1. **游戏引擎**（Unity/Unreal）只接受 Mesh
2. **3D 打印**需要封闭 Mesh
3. **影视渲染**（Blender/Maya）基于 Mesh 工作流
4. Mesh 支持 **UV 展开 → 纹理贴图 → PBR 材质**

所以 3D 生成的核心挑战是：**如何用可微分的方式生成高质量 Mesh？**

## 7.2 NeRF：用神经网络表示 3D 场景

### 7.2.1 核心思想

**NeRF（Neural Radiance Field，2020）** 用一个 MLP 网络来表示整个 3D 场景：

$$F_\theta: (\mathbf{x}, \mathbf{d}) \rightarrow (\mathbf{c}, \sigma)$$

- 输入：空间坐标 $\mathbf{x} = (x, y, z)$ + 观察方向 $\mathbf{d} = (\theta, \phi)$
- 输出：颜色 $\mathbf{c} = (r, g, b)$ + 体密度 $\sigma$（该点是否有物体）

### 7.2.2 体渲染（Volume Rendering）

给定一条从相机出发的射线 $\mathbf{r}(t) = \mathbf{o} + t\mathbf{d}$，最终像素颜色通过积分计算：

$$C(\mathbf{r}) = \int_{t_n}^{t_f} T(t) \cdot \sigma(\mathbf{r}(t)) \cdot \mathbf{c}(\mathbf{r}(t), \mathbf{d}) \, dt$$

其中透射率 $T(t) = \exp\left(-\int_{t_n}^{t} \sigma(\mathbf{r}(s)) \, ds\right)$。

实际实现中用离散采样近似：

$$\hat{C}(\mathbf{r}) = \sum_{i=1}^{N} T_i \cdot \alpha_i \cdot \mathbf{c}_i, \quad \alpha_i = 1 - \exp(-\sigma_i \delta_i)$$

```python
def volume_render(sigmas, colors, deltas):
    """
    NeRF 体渲染的核心实现
    sigmas: [N_rays, N_samples]  体密度
    colors: [N_rays, N_samples, 3]  颜色
    deltas: [N_rays, N_samples]  采样间距
    """
    # alpha = 1 - exp(-sigma * delta)
    alphas = 1.0 - torch.exp(-sigmas * deltas)

    # 透射率 T_i = prod_{j<i} (1 - alpha_j)
    T = torch.cumprod(1.0 - alphas + 1e-10, dim=-1)
    T = torch.cat([torch.ones_like(T[..., :1]), T[..., :-1]], dim=-1)

    # 加权求和
    weights = T * alphas  # [N_rays, N_samples]
    rgb = (weights.unsqueeze(-1) * colors).sum(dim=-2)  # [N_rays, 3]
    return rgb
```

### 7.2.3 NeRF 的局限

| 问题 | 说明 |
|------|------|
| **训练慢** | 单场景需要数小时 |
| **推理慢** | 每个像素需要数百次 MLP 前向 |
| **无法泛化** | 一个 NeRF = 一个场景，不能迁移 |
| **难以编辑** | 隐式表示无法直接修改几何 |
| **难以导出 Mesh** | 需要 Marching Cubes 后处理 |

## 7.3 3D Gaussian Splatting：实时可微渲染

### 7.3.1 核心思想

**3D Gaussian Splatting（3DGS, 2023）** 用大量**各向异性 3D 高斯椭球**来显式表示场景：

每个高斯由 5 个属性定义：
- **位置** $\boldsymbol{\mu} \in \mathbb{R}^3$：中心坐标
- **协方差** $\boldsymbol{\Sigma} \in \mathbb{R}^{3\times3}$：形状和朝向（用四元数 + 缩放参数化）
- **不透明度** $\alpha \in [0, 1]$：透明度
- **颜色** $\mathbf{c}$：用球谐函数（SH）表示视角相关颜色

$$G(\mathbf{x}) = \exp\left(-\frac{1}{2}(\mathbf{x} - \boldsymbol{\mu})^T \boldsymbol{\Sigma}^{-1} (\mathbf{x} - \boldsymbol{\mu})\right)$$

### 7.3.2 Splatting 渲染

与 NeRF 的"射线采样"不同，3DGS 采用 **Splatting（泼溅）** 策略：

```
NeRF：射线 → 沿射线采样 → 逐点查询 MLP → 积分
3DGS：高斯 → 投影到 2D → 按深度排序 → alpha 混合
```

渲染公式（前到后 alpha 混合）：

$$C = \sum_{i \in \mathcal{N}} c_i \cdot \alpha_i \cdot \prod_{j=1}^{i-1}(1 - \alpha_j)$$

### 7.3.3 3DGS vs NeRF

| 对比项 | NeRF | 3DGS |
|--------|------|------|
| 表示 | 隐式（MLP） | 显式（高斯点集） |
| 渲染速度 | ~1 FPS | **~200 FPS** |
| 训练时间 | 数小时 | **~10 分钟** |
| 可编辑性 | 差 | 好（直接操作高斯） |
| 内存 | 低 | 高（百万级高斯） |
| Mesh 导出 | Marching Cubes | SuGaR 等方法 |

## 7.4 SDS 优化：用 2D 扩散模型"蒸馏"出 3D

### 7.4.1 DreamFusion 的核心洞察

**DreamFusion（Google, 2022）** 提出了一个天才想法：**不需要 3D 训练数据，只用预训练的 2D 扩散模型就能生成 3D**。

核心思路：
1. 随机初始化一个 NeRF
2. 从随机视角渲染出 2D 图像
3. 用预训练的扩散模型评估"这张图像像不像文本描述的物体"
4. 将梯度反传回 NeRF，优化 3D 表示

### 7.4.2 Score Distillation Sampling（SDS）

SDS 的梯度公式：

$$\nabla_\theta \mathcal{L}_{SDS} = \mathbb{E}_{t, \epsilon}\left[ w(t) \left(\hat{\epsilon}_\phi(z_t; y, t) - \epsilon\right) \frac{\partial x}{\partial \theta} \right]$$

其中：
- $\theta$：NeRF 参数
- $x$：从 NeRF 渲染的图像
- $z_t$：对 $x$ 加噪后的潜变量
- $\hat{\epsilon}_\phi$：预训练扩散模型的噪声预测
- $y$：文本条件
- $w(t)$：时间步权重

```
┌─────────────────────────────────────────────┐
│           DreamFusion / SDS 优化流程          │
├─────────────────────────────────────────────┤
│                                              │
│  文本 "a cute cat"                           │
│       ↓                                      │
│  ┌──────────┐    随机视角     ┌──────────┐   │
│  │  NeRF θ  │ ──渲染──→     │ 2D 图像 x │   │
│  └────┬─────┘               └─────┬──────┘   │
│       ↑ 梯度反传                   ↓ 加噪     │
│       │                    ┌──────────────┐  │
│       │                    │ z_t = x + ε  │  │
│       │                    └──────┬───────┘  │
│       │                           ↓          │
│       │                  ┌────────────────┐  │
│       └── SDS 梯度 ──── │ 预训练扩散模型  │  │
│                          │ (Imagen/SD)    │  │
│                          └────────────────┘  │
│                                              │
│  重复数千次 → 3D 物体逐渐成形                  │
└─────────────────────────────────────────────┘
```

### 7.4.3 SDS 的问题

| 问题 | 表现 |
|------|------|
| **Janus Problem（多面问题）** | 物体每个面都长出正面特征（如每面都有脸） |
| **过度饱和** | 颜色过于鲜艳，缺乏真实感 |
| **过度平滑** | 细节丢失，纹理模糊 |
| **优化慢** | 单个物体需要 1-2 小时 |

后续改进：VSD（Variational SDS）、ISM（Interval Score Matching）等缓解了部分问题，但 SDS 范式本身的效率瓶颈难以根本解决。

## 7.5 多视角扩散：从 2D 到 3D 的桥梁

### 7.5.1 Zero-1-to-3：视角合成

**Zero-1-to-3（2023）** 的思路：先用扩散模型从单张图片生成多个视角的 2D 图像，再用多视角重建得到 3D。

$$p(x_{target} | x_{input}, \Delta\theta, \Delta\phi)$$

给定输入图像 $x_{input}$ 和目标视角变化 $(\Delta\theta, \Delta\phi)$，生成目标视角的图像。

### 7.5.2 多视角一致性问题

单独生成每个视角会导致**多视角不一致**——不同视角的图像无法拼成一个连贯的 3D 物体。

解决方案演进：

```
Zero-1-to-3（单视角独立生成）
    ↓ 不一致
SyncDreamer（多视角联合生成 + 3D 注意力）
    ↓ 改善但仍有瑕疵
Wonder3D（多视角 + 法线图联合生成）
    ↓
Instant3D / InstantMesh（大规模重建模型 LRM）
    ↓ 范式转变
直接在 3D 空间做扩散（Hunyuan3D-DiT）
```

## 7.6 大规模重建模型（LRM）

### 7.6.1 从优化到前馈

SDS 和多视角方法都需要**逐物体优化**，非常慢。**LRM（Large Reconstruction Model, 2023）** 提出了前馈范式：

$$\text{LRM}: \text{Image} \xrightarrow{\text{单次前向}} \text{3D Triplane}$$

核心架构：
1. **图像编码器**（DINOv2）：提取图像特征
2. **Transformer 解码器**：将图像特征解码为 Triplane 表示
3. **NeRF 渲染器**：从 Triplane 渲染多视角图像（训练时用）

```python
# LRM 伪代码
class LRM(nn.Module):
    def __init__(self):
        self.image_encoder = DINOv2()        # 冻结的图像编码器
        self.transformer = TransformerDecoder(
            num_layers=24,
            dim=1024,
            num_queries=3 * 32 * 32  # 3 个平面，每个 32×32
        )
        self.nerf_decoder = MLP(dim=64, out_dim=4)  # σ + RGB

    def forward(self, image):
        # 1. 编码图像
        features = self.image_encoder(image)  # [B, N, D]

        # 2. 解码为 Triplane
        triplane = self.transformer(features)  # [B, 3, 32, 32, C]

        # 3. 查询任意 3D 点
        # 将 (x,y,z) 投影到三个平面，双线性插值取特征，拼接后过 MLP
        return triplane
```

### 7.6.2 Triplane 表示

Triplane 是一种高效的 3D 表示：用三个正交平面（XY、XZ、YZ）的 2D 特征图来表示 3D 空间。

```
        Z
        ↑   ┌──────┐ YZ 平面
        │  /│      │
        │ / │      │
        │/  │      │
        ┼───┼──────┤──→ Y
       /│   │      │
      / │   └──────┘
     /  │  XZ 平面
    ↓   └──────────
    X   XY 平面

查询点 (x,y,z) 的特征：
f = f_xy(x,y) ⊕ f_xz(x,z) ⊕ f_yz(y,z)
```

优势：将 $O(N^3)$ 的体素存储压缩为 $O(3 \times N^2)$。

## 7.7 3D 原生扩散：直接在 3D 空间去噪

### 7.7.1 从"2D 蒸馏"到"3D 原生"

前面的方法（SDS、多视角扩散）本质上都是**借助 2D 扩散模型间接生成 3D**。更优雅的方案是：**直接在 3D 表示空间上训练扩散模型**。

| 方法 | 扩散空间 | 代表 |
|------|----------|------|
| 点云扩散 | 点的坐标 $(x,y,z)$ | Point-E (OpenAI) |
| 隐式场扩散 | SDF/NeRF 潜空间 | Shap-E (OpenAI) |
| Triplane 扩散 | Triplane 特征图 | LN3Diff, Rodin |
| **Flow-based 3D DiT** | **3D 隐式潜空间** | **Hunyuan3D** |

### 7.7.2 Point-E 与 Shap-E

**Point-E（OpenAI, 2022）**：
- 先用 DALL·E 2 生成一张图片
- 再用点云扩散模型从图片生成 3D 点云
- 快（~1 分钟）但质量一般

**Shap-E（OpenAI, 2023）**：
- 训练一个编码器将 3D 物体编码为隐式函数的参数
- 在参数空间上训练扩散模型
- 支持 Text→3D 和 Image→3D

```
Point-E:  Text → [DALL·E 2] → Image → [点云扩散] → Point Cloud
Shap-E:   Text → [参数空间扩散] → 隐式函数参数 → NeRF/Mesh
```

## 7.8 Hunyuan3D：Flow-based 3D DiT（重点）

### 7.8.1 为什么 Hunyuan3D 是当前 SOTA？

Hunyuan3D 2.0（腾讯，2025-01）在所有公开基准上超越了全部开源和闭源基线：

| 指标 | Hunyuan3D 2.0 | 次优方案 | 说明 |
|------|---------------|----------|------|
| CMMD ↓ | **3.193** | 4.5+ | 越低越好，衡量生成质量 |
| FID ↓ | **49.165** | 60+ | 越低越好，衡量分布距离 |
| CLIP-score ↑ | **0.809** | 0.78 | 越高越好，衡量语义一致性 |

### 7.8.2 两阶段架构

Hunyuan3D 的核心设计是**形状与纹理解耦**：

```
┌──────────────────────────────────────────────┐
│         Hunyuan3D 2.0 — 两阶段 3D 生成        │
├──────────────────────────────────────────────┤
│                                               │

│       ↓                                       │
│  ╔════════════════════════════════════════╗   │
│  ║  Stage 1: Hunyuan3D-DiT（形状生成）     ║   │
│  ║                                        ║   │
│  ║  DINOv2 Image Encoder                  ║   │
│  ║       ↓ 条件特征                        ║   │
│  ║  Flow-based DiT                        ║   │
│  ║  （在 3D 隐式潜空间做 Flow Matching）    ║   │
│  ║       ↓                                ║   │
│  ║  3D 隐式表示 → Marching Cubes → Mesh   ║   │
│  ╚════════════════╤═══════════════════════╝   │
│                   ↓ 裸 Mesh（无纹理）          │
│  ╔════════════════════════════════════════╗   │
│  ║  Stage 2: Hunyuan3D-Paint（纹理合成）   ║   │
│  ║                                        ║   │
│  ║  多视角渲染 → 2D 扩散先验上色            ║   │
│  ║       ↓                                ║   │
│  ║  多视角纹理 → 反投影融合 → UV 贴图       ║   │
│  ╚════════════════╤═══════════════════════╝   │
│                   ↓                           │
│  输出：高分辨率纹理 3D 资产（GLB/OBJ）         │
└──────────────────────────────────────────────┘
```

### 7.8.3 Stage 1：Flow-based 3D DiT

这是 Hunyuan3D 的核心创新。回顾第 6 章的 Flow Matching：

**传统 DDPM**：离散时间步，逐步加噪/去噪
**Flow Matching**：连续时间 $t \in [0, 1]$，学习最优传输路径

$$x_t = (1 - t) \cdot x_0 + t \cdot \epsilon, \quad t \in [0, 1]$$

训练目标——预测速度场：

$$\mathcal{L}_{FM} = \mathbb{E}_{t, x_0, \epsilon} \left\| v_\theta(x_t, t, c) - (\epsilon - x_0) \right\|^2$$

在 Hunyuan3D 中：
- $x_0$ 不是 2D 图像的潜变量，而是 **3D 隐式表示的潜变量**
- $c$ 是 DINOv2 编码的图像条件特征
- $v_\theta$ 是 DiT（Diffusion Transformer）网络

```python
# Hunyuan3D-DiT 训练伪代码
class Hunyuan3DDiT(nn.Module):
    def __init__(self):
        self.image_encoder = DINOv2()  # 冻结
        self.dit = DiTBlock(
            dim=1024,
            num_heads=16,
            num_layers=28,
        )
        self.shape_decoder = ShapeDecoder()  # 隐式表示 → Mesh

    def forward(self, x_0, image_cond):
        """
        x_0: 3D 隐式潜变量 [B, L, D]
        image_cond: 输入图片
        """
        # 1. 编码条件
        cond = self.image_encoder(image_cond)  # [B, N, D]

        # 2. Flow Matching 采样时间和噪声
        t = torch.rand(x_0.shape[0], device=x_0.device)
        eps = torch.randn_like(x_0)

        # 3. 插值得到 x_t
        x_t = (1 - t[:, None, None]) * x_0 + t[:, None, None] * eps

        # 4. DiT 预测速度场
        v_pred = self.dit(x_t, t, cond)

        # 5. Flow Matching 损失
        v_target = eps - x_0
        loss = F.mse_loss(v_pred, v_target)
        return loss

    @torch.no_grad()
    def generate(self, image_cond, num_steps=50):
        """推理：从噪声出发，沿速度场积分"""
        cond = self.image_encoder(image_cond)
        x = torch.randn(1, L, D)  # 纯噪声

        dt = 1.0 / num_steps
        for i in range(num_steps):
            t = torch.tensor([1.0 - i * dt])
            v = self.dit(x, t, cond)
            x = x - v * dt  # 欧拉积分

        # 解码为 Mesh
        mesh = self.shape_decoder(x)
        return mesh
```

### 7.8.4 为什么用 DiT 而不是 U-Net？

| 对比项 | U-Net | DiT |
|--------|-------|-----|
| 归纳偏置 | 局部卷积 | 全局注意力 |
| 3D 适配 | 需要 3D 卷积（内存爆炸） | Token 序列天然支持任意维度 |
| 条件注入 | AdaGN / 交叉注意力 | AdaLN-Zero + 交叉注意力 |
| 扩展性 | 受限于卷积核 | Scaling Law 友好 |

DiT 将 3D 隐式表示展平为 token 序列，用自注意力捕捉全局 3D 结构，用交叉注意力注入图像/文本条件。这与第 6 章介绍的 2D DiT 架构一脉相承。

### 7.8.5 Stage 2：Hunyuan3D-Paint（纹理合成）

拿到裸 Mesh 后，第二阶段负责"上色"：

1. **多视角渲染**：将裸 Mesh 从 N 个视角渲染为 2D 图像
2. **2D 扩散先验**：用类似 Stable Diffusion 的模型为每个视角生成纹理
3. **反投影融合**：将多视角纹理反投影回 Mesh 表面，融合为统一的 UV 贴图

```
裸 Mesh → 多视角渲染 → N 张灰色图
                         ↓
              2D 扩散模型上色 → N 张彩色图
                         ↓
              反投影到 Mesh → UV 纹理贴图
```

### 7.8.6 Hunyuan3D 2.1：PBR 材质生成

Hunyuan3D 2.1（2025-06）在 2.0 基础上新增了 **PBR（Physically-Based Rendering）材质生成**：

| 贴图类型 | 作用 | 说明 |
|----------|------|------|
| **Albedo Map** | 基础色 | 物体本身的颜色，不含光照信息 |
| **Roughness Map** | 粗糙度 | 0=镜面反射，1=完全漫反射 |
| **Metallic Map** | 金属度 | 0=非金属（塑料），1=金属 |
| **Normal Map** | 法线贴图 | 表面微观凹凸细节 |

```
                    ┌─────────────────┐
                    │  Albedo（基础色） │ → 物体颜色
                    ├─────────────────┤
裸 Mesh ──Paint──→ │  Roughness（粗糙）│ → 光泽/哑光
                    ├─────────────────┤
                    │  Metallic（金属） │ → 金属/非金属
                    ├─────────────────┤
                    │  Normal（法线）   │ → 微观凹凸
                    └─────────────────┘
                            ↓
                    可直接导入 Unity / Unreal Engine
```

有了 PBR 材质，生成的 3D 资产在不同光照环境下都能正确渲染，达到**工业级可用**标准。

## 7.10 3D 生成的核心挑战与未来方向

### 7.10.1 当前挑战

1. **3D 训练数据稀缺**：高质量 3D 数据集（Objaverse）仅百万级，远不及 2D 图像的十亿级
2. **多视角一致性**：从 2D 生成 3D 时，不同视角的一致性仍是难题
3. **拓扑复杂物体**：镂空、薄壁、多连通等复杂拓扑结构生成困难
4. **动态 3D**：带骨骼绑定和动画的 3D 资产生成仍处于早期
5. **场景级生成**：从单物体到完整场景的跨越

### 7.10.2 未来方向

```
当前（2025）                          未来（2026+）
─────────────────────────────────────────────────
单物体生成                    →    场景级 3D 生成
静态 Mesh                    →    带骨骼 + 动画的角色
PBR 材质                     →    物理仿真材质（布料、流体）
Image/Text → 3D              →    Video → 4D（时空重建）
独立生成                      →    3D 世界模型（可交互）
```

### 7.10.3 与大语言模型的交汇

3D 生成正在与 LLM 生态深度融合：

- **LLM 作为 3D 场景编排器**：用自然语言描述场景布局，LLM 调用 3D 生成 API 组装
- **多模态理解**：GPT-4o / Gemini 已能理解 3D 空间关系
- **具身智能**：3D 世界模型 + LLM 规划 = 机器人的"大脑"
- **游戏/影视自动化**：文本 → 剧本 → 3D 场景 → 渲染 → 成片
